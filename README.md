# 🐹 Go Learning Vault

Личный репозиторий для изучения **Go**: теория, лабораторные работы, практические проекты, ошибки и полезные кодовые рецепты.

Здесь постепенно собирается база знаний по Go-разработке: от базового синтаксиса и настройки окружения до практических кейсов, тестов, backend-разработки и микросервисной архитектуры.

---

## 📌 Что находится в репозитории

```text
GO/
├── Library/      # Obsidian-хранилище с заметками
├── Projects/     # Go-проекты и лабораторные работы
├── README.md
├── .gitignore
└── .gitattributes
```

---

## 🧠 Library

`Library` — это Obsidian-хранилище с теорией, заметками и материалами по обучению.

Внутри хранятся:

* 📚 теория по Go;
* 🧩 заметки по синтаксису и правилам языка;
* 🧪 материалы лабораторных работ;
* 🛠️ кодовые рецепты;
* 🐞 разборы ошибок;
* 🔗 связанные темы и ссылки;
* 📝 шаблоны для новых заметок.

Основная идея простая: **теория отдельно, практика отдельно**.

---

## 💻 Projects

`Projects` — это папка с практическими Go-проектами, лабораторными работами и учебными кейсами разработки.

Каждая новая лабораторная или практический кейс создаётся в отдельной папке:

```text
Projects/
├── 000_Go_Environment_Setup/
├── 001_Basic_Syntax/
├── 002_Functions/
├── 003_Structs_And_Methods/
├── 004_Interfaces/
└── ...
```

Внутри конкретного проекта могут находиться:

* исходный код;
* тесты;
* `go.mod`;
* `go.sum`;
* локальный `README.md`;
* примеры запуска;
* конфигурационные файлы;
* вспомогательные материалы для выполнения лабораторной.

---

## 🗂️ Как устроены лабораторные в Obsidian

Для каждой лабораторной используется единая структура:

```text
02_Practice/01_Labs/XXX_Lab_Name/
├── 00_Lab Card.md
├── 01_Task.md
├── 02_Worklog.md
├── 03_Code.md
├── 04_Errors.md
└── 05_Result.md
```

### Что фиксируется в лабораторной

* 🎯 цель;
* 📋 задание;
* 🧭 ход выполнения;
* 🧱 структура проекта;
* 💻 команды;
* 🧩 ключевой код;
* 🐞 ошибки;
* 🔍 причины ошибок;
* ✅ решения;
* 🏁 итог;
* 🔗 связанная теория.

---

## 📚 Разделение знаний

Репозиторий строится по принципу чистого графа Obsidian:

```text
01_Theory   → теория, правила, концепции, паттерны
02_Practice → лабораторные, кейсы, ошибки, решения
```

### Правила

* 🧠 теория не смешивается с практикой;
* 🧪 лабораторные могут ссылаться на теорию;
* 🐞 важные ошибки выносятся в debug diary;
* 🛠️ универсальные решения выносятся в code recipes;
* 🧼 в репозиторий не добавляются IDE, пакеты, архивы и временные файлы.

---

## 🚫 Что не хранится в репозитории

В репозиторий не добавляются:

* IDE-файлы;
* временные файлы;
* кэши;
* архивы;
* установленные пакеты;
* `.exe`, `.dll`, `.zip`;
* GoLand, плагины и тулзы;
* локальное состояние Obsidian;
* сборочные артефакты.

Репозиторий должен оставаться чистым: только знания, код лабораторных и нужные материалы.

## 🔄 Рабочий процесс

Обычно процесс такой:

1. Изучается новая тема.
2. Теория фиксируется в Obsidian.
3. Практическая часть выполняется в `Projects`.
4. Ход выполнения записывается в worklog.
5. Ошибки и решения выносятся отдельно.
6. Лабораторная связывается с теоретическими заметками.
7. Изменения коммитятся в GitHub.

Примеры коммитов:

```text
Add Go environment setup lab
Add variables and types lab
Update Go modules theory
Add debug note for GoLand unresolved symbols
Add Go CLI commands note
Update lab worklog
Refactor lab folder structure
```

## 🎯 Цель репозитория

Цель этого репозитория — собрать аккуратную и понятную базу знаний по Go.

Он нужен, чтобы:

* видеть прогресс обучения;

* быстро возвращаться к ошибкам и их причинам;

* постепенно собирать личную базу Go-разработчика.
