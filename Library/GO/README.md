## Принцип структуры

- `00_Course` — маршрут обучения.
- `01_Theory` — синтаксис, правила, концепции, паттерны, лайфхаки.
- `02_Practice` — лабораторные, мини-кейсы, ошибки, решения, кодовые рецепты.
- `99_Templates` — шаблоны заметок.

## Правило работы

Один чат = одна лабораторная или один практический кейс.

В начале чата указывается:

- номер лабы;
- тема;
- цель;
- задание;
- ожидаемый результат;
- технологии;
- наличие кода, ошибок или скриншотов.

По итогам лабы используется команда:

> собери файлы для Obsidian
