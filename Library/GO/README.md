## Принцип структуры

- `00_Start_Here` — точка входа для работы с хранилищем.
- `00_Vault_Structure` — карта папок и правила размещения заметок.
- `00_Course` — маршрут обучения по модулям.
- `00_Roadmaps` — дорожные карты, текущий фокус и пробелы.
- `01_Theory` — теория по маршруту от Go basics до microservices и advanced Go.
- `02_Practice` — лабораторные, мини-кейсы, ошибки, решения, кодовые рецепты.
- `03_Reference` — ссылки, глоссарий и справочные материалы.
- `99_Templates` — рабочие шаблоны заметок.

## Теория

- `01_Go_Basics` — базовый синтаксис и типы.
- `02_Go_Tooling` — команды Go и структура проекта.
- `03_CLI_And_Files` — CLI, файлы, JSON, конфиги.
- `04_Stdlib` — стандартная библиотека.
- `05_HTTP_Backend` — HTTP, REST, middleware, JSON API.
- `06_Databases` — PostgreSQL, SQL, migrations, repository.
- `07_Concurrency` — goroutines, channels, context, worker pool.
- `08_Architecture` — слои, DTO, DI, Clean Architecture.
- `09_Production` — config, secrets, logging, healthcheck.
- `10_Microservices` — gateway, discovery, broker, tracing.
- `11_Advanced_Go` — testing, benchmarks, error wrapping, pprof.
- `90_Rules`, `91_Lifehacks`, `99_Aliases`, `99_Atomic_Go` — поддерживающие разделы.

## Правило работы

Один чат = одна лабораторная или один практический кейс.

В начале чата указывается:

- номер лабы;
- тема;
- цель;
- задание;
- ожидаемый результат;
- технологии;
- наличие кода, ошибок или скриншотов.

По итогам лабы используется команда:

> собери файлы для Obsidian
