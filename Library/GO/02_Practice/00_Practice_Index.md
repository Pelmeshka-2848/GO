# Practice Index

Канонический индекс раздела практики.

Практика — это всё, что отвечает на вопрос: **как я это сделал в конкретной работе или кейсе**.

## Разделы практики

| Раздел | Смысл |
|---|---|
| [[02_Practice/01_Labs/00_Labs_Index|Лабораторные]] | учебные лабораторные и треки по Go |
| [[02_Practice/02_Mini_Cases/00_Mini_Cases_Index|Мини-кейсы]] | короткие самостоятельные задачи |
| [[02_Practice/02_Development_Cases/Development Cases Index|Кейсы разработки]] | API, CRUD, auth, архитектура |
| [[02_Practice/03_Debug_Diary/00_Debug_Diary_Index|Дневник ошибок]] | ошибки, баги, проблемы окружения |
| [[02_Practice/04_Code_Recipes/00_Code_Recipes_Index|Кодовые рецепты]] | прикладные рецепты кода |
| [[02_Practice/05_Project_Notes/Project Notes Index|Заметки проектов]] | конспекты по своим проектам |

## Как выбрать формат практики

| Если материал про... | Куда писать |
|---|---|
| большую учебную работу | `02_Practice/01_Labs` |
| короткую задачу на закрепление | `02_Practice/02_Mini_Cases` |
| прикладной backend-сценарий | `02_Practice/02_Development_Cases` |
| ошибку и ее решение | `02_Practice/03_Debug_Diary` |
| переиспользуемый прием кода | `02_Practice/04_Code_Recipes` |
| заметки по своему проекту | `02_Practice/05_Project_Notes` |

## Что сюда фиксировать

- лабораторные и мини-кейсы;
- команды и проверки;
- структуру учебных проектов;
- ключевые фрагменты уже написанного кода;
- ошибки, причины и решения;
- итоговые результаты.

## Связь с теорией

В каждой лабораторной или кейсе должен быть блок:

```markdown
## Связанная теория

- [[01_Theory/01_Go_Basics/00_Index]]
- [[01_Theory/02_Go_Tooling/00_Index]]
```

Так граф не превращается в кашу: практика ссылается на нужную теорию, но теория не тащит на себя все практические заметки.

Подробная карта папок: [[00_Vault_Structure]].
