# 05 — Результат

## Итог лабораторной

В ходе выполнения лабораторной работы было подготовлено базовое окружение разработки на Go.

Были выполнены следующие действия:

- установлен Go SDK;
- проверена доступность команды `go`;
- проверены основные параметры окружения Go;
- настроен Go SDK в GoLand;
- создана рабочая директория для учебных Go-проектов;
- создан минимальный Go-модуль;
- написана программа `main.go`;
- добавлен минимальный тест `main_test.go`;
- выполнено форматирование кода;
- выполнен запуск программы;
- выполнен запуск тестов;
- выполнена сборка исполняемого файла.

## Контрольные результаты

| Проверка            | Команда                          | Ожидаемый результат                           | Статус |
| ------------------- | -------------------------------- | --------------------------------------------- | ------ |
| Проверка версии Go  | `go version`                     | вывод версии Go                               | ==⬜==  |
| Проверка пути до Go | `where.exe go`                   | путь к `go.exe`                               | ==⬜==  |
| Проверка окружения  | `go env ...`                     | значения `GOROOT`, `GOPATH`, `GOOS`, `GOARCH` | ==⬜==  |
| Создание модуля     | `go mod init ...`                | создан `go.mod`                               | ==⬜==  |
| Форматирование      | `go fmt ./...`                   | код отформатирован                            | ==⬜==  |
| Запуск              | `go run .`                       | программа вывела текст                        | ==⬜==  |
| Тесты               | `go test ./...`                  | тесты прошли успешно                          | ==⬜==  |
| Сборка              | `go build -o bin\hello-go.exe .` | создан `.exe`                                 | ==⬜==  |
| Запуск бинарника    | `.\bin\hello-go.exe`             | бинарник запустился                           | ==⬜==  |

## Практический вывод

Окружение Go считается готовым, если проект проходит полный минимальный цикл:

```text
go mod init
→ go fmt
→ go run
→ go test
→ go build
```

Этот цикл является базовым workflow для следующих лабораторных работ и практических кейсов.
