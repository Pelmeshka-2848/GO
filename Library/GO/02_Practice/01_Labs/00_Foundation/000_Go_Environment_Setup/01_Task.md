# 01 — Задание

## Тема

Настройка окружения разработки для языка Go.

## Цель задания

Подготовить рабочую среду для дальнейшего изучения Go и выполнения лабораторных работ, кейсов разработки и микросервисных задач.

## Исходные условия

- Основная ОС: Windows.
- Основная IDE: GoLand.
- Терминал: PowerShell.
- Docker в этой лабораторной не используется.
- PostgreSQL в рамках этой лабораторной не требуется.
- Основная задача — проверить саму Go toolchain и базовый workflow проекта.

## Задание

Выполнить следующие действия:

1. Установить Go SDK.
2. Проверить доступность команды `go` из PowerShell.
3. Проверить основные параметры окружения через `go env`.
4. Настроить Go SDK в GoLand.
5. Создать отдельную директорию для учебных Go-проектов.
6. Создать минимальный Go-проект `hello-go`.
7. Инициализировать Go-модуль через `go mod init`.
8. Написать минимальную программу `main.go`.
9. Отформатировать код через `go fmt`.
10. Запустить программу через `go run`.
11. Добавить минимальный тест.
12. Проверить проект через `go test`.
13. Собрать исполняемый файл через `go build`.
14. Зафиксировать типовые ошибки и способы их устранения.

## Проверочные команды

```powershell
go version
where.exe go
go env GOROOT GOPATH GOMODCACHE GOPROXY GOOS GOARCH
```

```powershell
go mod init example.com/sheve/hello-go
go fmt ./...
go run .
go test ./...
go build -o bin\hello-go.exe .
.\bin\hello-go.exe
```

## Критерии выполнения

Лабораторная считается выполненной, если:

- PowerShell успешно выводит версию Go;
- `where.exe go` показывает путь к исполняемому файлу `go.exe`;
- GoLand видит установленный Go SDK;
- в проекте создан файл `go.mod`;
- программа запускается командой `go run .`;
- тесты выполняются командой `go test ./...`;
- проект собирается в `.exe`-файл;
- ошибки и решения зафиксированы в `04_Errors.md`.

## Что не входит в эту лабораторную

- Docker.
- PostgreSQL.
- REST API.
- HTTP-сервер.
- Микросервисная архитектура.
- CI/CD.
- Работа с внешними Go-библиотеками.
