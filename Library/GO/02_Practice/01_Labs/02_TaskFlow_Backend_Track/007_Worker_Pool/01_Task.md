# 01. Task — Worker Pool

## Тема

Worker Pool

## Цель

Реализовать worker pool для параллельной обработки задач.

## Исходные данные

- Есть набор независимых задач, которые можно выполнять параллельно.
- Количество задач может быть больше количества workers.
- Нужно ограничить параллелизм, чтобы не запускать неограниченное число goroutines.

## Задание

1. Описать модель задачи и результата.
2. Создать фиксированное количество workers.
3. Передавать задачи workers через канал.
4. Собрать результаты обработки.
5. Корректно завершить workers после закрытия очереди задач.
6. Зафиксировать, как проверялось отсутствие зависаний и потери результатов.

## Ожидаемый результат

Получается учебная реализация worker pool: задачи распределяются между workers, параллелизм ограничен, результаты собираются в одном месте, завершение происходит предсказуемо.
