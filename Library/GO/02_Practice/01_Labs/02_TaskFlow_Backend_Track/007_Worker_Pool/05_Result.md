# 05. Result — Worker Pool

## Итог

Лабораторная закрепляет паттерн worker pool: ограниченный набор goroutines обрабатывает очередь задач, а программа контролирует сбор результатов и завершение.

## Что было сделано

- Описана задача на ограничение параллелизма.
- Зафиксированы ключевые элементы worker pool.
- Подготовлены критерии проверки корректного завершения.

## Что было изучено

- Зачем ограничивать количество goroutines.
- Как channels помогают передавать задачи и результаты.
- Почему нужно явно продумывать завершение workers.
- Где возникают deadlock и data race.

## Вывод

Цель лабораторной считается достигнутой, когда worker pool обрабатывает все задачи, не зависает, не теряет результаты и не запускает больше workers, чем задано.
