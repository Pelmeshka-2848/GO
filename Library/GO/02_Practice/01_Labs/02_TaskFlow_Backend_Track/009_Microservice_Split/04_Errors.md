# Lab 009 - Microservice Split — Errors

| Слишком раннее разделение на сервисы | Нет реальной боли монолита | Сначала описать границы и оставить простой монолит, если он решает задачу | [[01_Theory/08_Architecture/Monolith]] |
| Общая база для всех сервисов | Сервисы остаются связанными через данные | Определить владельца данных для каждого сервиса | [[01_Theory/10_Microservices/What Is Microservice]] |
| Нет наблюдаемости | Сложно понять, где сломался распределенный запрос | Планировать logs, metrics и tracing до усложнения системы | [[01_Theory/10_Microservices/Distributed Tracing]] |
