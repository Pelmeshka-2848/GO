# Lab 009 - Microservice Split — Task

## Задание

Разобрать, как монолитный TaskFlow backend можно разделить на несколько сервисов. Лабораторная не требует production-реализации микросервисов; цель — спроектировать границы сервисов, контракты и риски такого разделения.

Минимальный результат:

- определить возможные сервисы;
- описать ответственность каждого сервиса;
- указать, какие данные принадлежат каждому сервису;
- описать способы взаимодействия между сервисами.

## Ограничения

- Не дробить приложение только ради дробления.
- Не смешивать ответственность разных сервисов.
- Учитывать сетевые ошибки и временную недоступность сервисов.
- Зафиксировать, какие части пока разумнее оставить в монолите.

## Критерии готовности

- [ ] Описаны границы минимум двух сервисов.
- [ ] Зафиксированы контракты взаимодействия.
- [ ] Перечислены риски разделения на микросервисы.
- [ ] Добавлены ссылки на связанную теорию по архитектуре и микросервисам.
