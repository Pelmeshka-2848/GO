# Lab 009 - Microservice Split — Worklog

## Ход работы

### Шаг 1. Анализ монолита

- выделить основные доменные области TaskFlow;
- найти зависимости между модулями;
- определить данные, которыми владеет каждая область;
- отметить места, где разделение усложнит систему.

### Шаг 2. Проектирование границ сервисов

- описать сервис задач;
- описать сервис пользователей или авторизации;
- определить, какие API нужны между сервисами;
- отметить синхронные и асинхронные взаимодействия.

## Что получилось

- Сформирована карта возможного разделения TaskFlow backend.
- Зафиксированы риски микросервисного подхода.

## Что не получилось

- Реальная реализация нескольких сервисов не входит в минимальный разбор.
- Service discovery, broker и tracing стоит разбирать отдельными лабораторными.
