# Lab 004 - REST CRUD API — Result

## Проверка

- [ ] API запускается локально.
- [ ] CRUD-операции проверены вручную.
- [ ] Ошибочные запросы возвращают понятный статус и JSON-ответ.

## Итог

В ходе выполнения работы была разобрана структура простого REST CRUD API: маршруты, HTTP-методы, JSON-обмен и базовая обработка ошибок.

## Что стало понятнее

- REST — это соглашение о работе с ресурсами через HTTP.
- CRUD-операции удобно сопоставлять с HTTP-методами.
- Даже учебный API должен явно различать успешные и ошибочные ответы.
