# Lab 004 - REST CRUD API — Task

## Задание

Разработать учебный REST CRUD API для одной сущности. Лабораторная должна закрепить базовую работу с HTTP-методами, маршрутами, JSON-запросами и JSON-ответами.

Минимальный набор операций:

- создать запись через `POST`;
- получить список записей через `GET`;
- получить одну запись по идентификатору через `GET`;
- обновить запись через `PUT` или `PATCH`;
- удалить запись через `DELETE`.

## Ограничения

- Хранение данных может быть in-memory, без базы данных.
- Ответы должны быть в JSON-формате.
- Ошибки клиента должны возвращать подходящие HTTP-статусы.
- Логика должна оставаться учебной и простой, без production-усложнений.

## Критерии готовности

- [ ] Все CRUD-операции описаны и проверены вручную.
- [ ] Для успешных и ошибочных сценариев указаны ожидаемые HTTP-статусы.
- [ ] В worklog зафиксированы команды запуска и проверки.
- [ ] Добавлены ссылки на связанную теорию по HTTP и JSON.
