# Lab 004 - REST CRUD API — Worklog

## Ход работы

### Шаг 1. Подготовка проекта

```bash
go mod init <module-name>
go run .
```

### Шаг 2. Описание API

- определить сущность и ее поля;
- описать маршруты CRUD;
- связать HTTP-методы с операциями;
- продумать ответы для успешных и ошибочных сценариев.

### Шаг 3. Проверка вручную

- проверить создание записи;
- проверить получение списка;
- проверить получение записи по идентификатору;
- проверить обновление;
- проверить удаление;
- отдельно проверить ошибочные запросы.

## Что получилось

- Зафиксирован учебный сценарий REST CRUD API.
- Понятно, какие HTTP-методы отвечают за каждую CRUD-операцию.
- Определены точки проверки для ручного тестирования.

## Что не получилось

- Пока не фиксировалась полноценная реализация кода.
- Автоматические тесты можно добавить отдельной лабораторной.
