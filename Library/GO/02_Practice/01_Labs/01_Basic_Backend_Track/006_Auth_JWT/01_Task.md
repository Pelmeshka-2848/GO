# Lab 006 - Auth JWT — Task

## Задание

Добавить в учебный API базовую авторизацию через JWT. Лабораторная должна закрепить разделение публичных и защищенных маршрутов, выдачу токена и проверку токена в middleware.

Минимальный сценарий:

- пользователь отправляет данные для входа;
- сервер возвращает JWT;
- защищенный маршрут принимает запрос только с валидным токеном;
- запрос без токена или с неверным токеном получает ошибку авторизации.

## Ограничения

- Не хранить секрет JWT в открытом виде как реальный production-секрет.
- Пароли в учебной заметке описывать концептуально, без реальных данных.
- JWT не должен заменять проверку прав доступа, если появятся роли.
- Ошибки авторизации должны возвращать понятный HTTP-статус.

## Критерии готовности

- [ ] Описан login-сценарий.
- [ ] Описан защищенный маршрут.
- [ ] Зафиксированы варианты ошибок: нет токена, неверный токен, истекший токен.
- [ ] Добавлены ссылки на теорию по JWT, middleware и HTTP-статусам.
