# Lab 006 - Auth JWT — Result

## Проверка

- [ ] Login возвращает JWT.
- [ ] Защищенный маршрут доступен с валидным токеном.
- [ ] Запрос без токена получает ошибку авторизации.
- [ ] Запрос с неверным токеном получает ошибку авторизации.

## Итог

В ходе выполнения работы был разобран базовый механизм авторизации через JWT: выдача токена, передача токена клиентом и проверка токена перед доступом к защищенному маршруту.

## Что стало понятнее

- JWT — это подписанный токен, а не сессия на сервере.
- Middleware удобно использовать для общей проверки авторизации.
- Ошибки `401` и `403` нужно различать по смыслу.
