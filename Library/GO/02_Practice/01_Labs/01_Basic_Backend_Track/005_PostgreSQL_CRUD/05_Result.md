# Lab 005 - PostgreSQL CRUD — Result

## Проверка

- [ ] Приложение подключается к PostgreSQL.
- [ ] Таблица создана или миграция применена.
- [ ] CRUD-операции проходят через базу, а не in-memory хранилище.

## Итог

В ходе выполнения работы REST CRUD был связан с PostgreSQL. Основной акцент — на подключении к базе, параметризованных SQL-запросах и разделении HTTP-логики от слоя хранения.

## Что стало понятнее

- База данных добавляет отдельный слой ошибок.
- Строку подключения нужно оформлять аккуратно и не смешивать с кодом.
- Repository помогает не превращать handlers в смесь HTTP и SQL.
