# Lab 003 - HTTP Server — Worklog

## План выполнения

1. Создать учебный Go-проект.
2. Подготовить `main.go`.
3. Добавить первый handler.
4. Зарегистрировать маршруты.
5. Запустить сервер на локальном порту.
6. Проверить ответы endpoint-ов.

## Команды

```bash
go mod init example/http-server
go run .
curl http://localhost:8080/
curl http://localhost:8080/health
```

## Проверки

- Сервер стартует без panic.
- Корневой endpoint возвращает ответ.
- Health endpoint возвращает успешный статус.
- При занятом порте ошибка понятна и зафиксирована.

## Заметки по ходу

- Фактический порт записать после выполнения.
- Если возникла ошибка занятого порта, вынести ее в `04_Errors.md`.
