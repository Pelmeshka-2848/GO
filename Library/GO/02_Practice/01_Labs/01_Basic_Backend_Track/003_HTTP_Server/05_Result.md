# Lab 003 - HTTP Server — Result

## Проверка

- [ ] HTTP-сервер запускается локально.
- [ ] Healthcheck или базовый endpoint возвращает успешный ответ.
- [ ] Handler получает запрос и формирует ответ.
- [ ] Ошибочные маршруты обрабатываются предсказуемо.

## Итог

В ходе выполнения работы был разобран минимальный HTTP-сервер на Go: запуск сервера, регистрация handler и проверка ответа через браузер или HTTP-клиент.

## Что стало понятнее

- HTTP handler — это точка входа для обработки запроса.
- Сервер и маршруты лучше проверять маленькими шагами.
- Даже простой endpoint полезно использовать как проверку, что приложение живо.
