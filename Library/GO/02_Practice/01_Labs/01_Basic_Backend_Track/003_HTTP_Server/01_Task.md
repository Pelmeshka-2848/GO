# Lab 003 - HTTP Server — Task

## Цель

Научиться запускать минимальный HTTP-сервер на Go и понимать роль handler-а.

## Задание

Создать учебный HTTP-сервер с несколькими простыми endpoint-ами:

- корневой путь для проверки, что сервер работает;
- endpoint здоровья;
- один endpoint с текстовым или JSON-ответом.

## Ограничения

- Использовать стандартный пакет `net/http`.
- Не подключать базу данных.
- Не усложнять архитектуру раньше времени.
- Явно зафиксировать порт запуска.

## Критерии готовности

- [ ] Сервер запускается локально.
- [ ] Endpoint-ы отвечают ожидаемыми статусами.
- [ ] Проверка выполнена через браузер, `curl` или аналог.
- [ ] Команды и результаты записаны в `02_Worklog.md`.
