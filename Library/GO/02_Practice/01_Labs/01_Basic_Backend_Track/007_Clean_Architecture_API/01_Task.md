# Lab 007 - Clean Architecture API — Task

## Задание

Перестроить учебный REST API так, чтобы HTTP-слой, бизнес-логика и слой хранения были разделены. Лабораторная должна закрепить идею Clean Architecture на небольшом backend-примере.

Минимальные слои:

- handler или controller для HTTP;
- service или usecase для бизнес-логики;
- repository для работы с хранилищем;
- domain/entity для основных моделей.

## Ограничения

- Не усложнять структуру раньше, чем появляется реальная причина.
- Domain-слой не должен зависеть от HTTP.
- Service не должен знать детали `http.Request` и `http.ResponseWriter`.
- Repository не должен содержать бизнес-правила.

## Критерии готовности

- [ ] Описана структура папок.
- [ ] Для каждого слоя указана ответственность.
- [ ] Показано направление зависимостей.
- [ ] Зафиксированы типичные ошибки разделения слоев.
