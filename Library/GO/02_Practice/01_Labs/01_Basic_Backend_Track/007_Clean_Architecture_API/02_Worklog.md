# Lab 007 - Clean Architecture API — Worklog

## Ход работы

### Шаг 1. Разбор текущей структуры

- найти, где смешаны HTTP, SQL и бизнес-логика;
- определить сущности домена;
- выделить операции usecase/service;
- определить интерфейсы repository.

### Шаг 2. Проверка запуска

```bash
go run .
```

### Шаг 3. Проверка связей между слоями

- handler вызывает service;
- service вызывает repository через интерфейс;
- repository работает с конкретным хранилищем;
- domain-модели не зависят от транспорта.

## Что получилось

- Описан учебный переход от простого REST API к слоистой архитектуре.
- Зафиксированы ответственности основных слоев.

## Что не получилось

- Полная реализация структуры должна фиксироваться после самостоятельной практики.
- Продвинутые варианты DI и composition root можно разобрать отдельно.
