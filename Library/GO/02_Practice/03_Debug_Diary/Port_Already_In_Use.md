# Порт уже занят

## Контекст

Возникает при запуске HTTP-сервера, если выбранный порт уже занят другим процессом.

## Текст ошибки

```text
address already in use
listen tcp :8080: bind: address already in use
```

## Причина

На этом порту уже работает другое приложение или предыдущий запуск сервера не был остановлен.

## Решение

- Найти процесс, который занимает порт.
- Остановить старый процесс или выбрать другой порт.
- Проверить, что приложение не запускается дважды из IDE и терминала одновременно.

## Как не повторить

- Перед повторным запуском останавливать предыдущий сервер.
- Для лабораторных фиксировать используемый порт в worklog.
- Для нескольких сервисов заранее выделять разные порты.
