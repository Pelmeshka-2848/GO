# Data race в конкурентном коде

## Контекст

Возникает в конкурентном коде, когда несколько goroutines одновременно читают и изменяют одни и те же данные без синхронизации.

## Текст ошибки

```text
WARNING: DATA RACE
```

## Причина

Общее состояние используется из нескольких goroutines без mutex, channel-синхронизации или другого механизма защиты.

## Решение

- Найти общий ресурс, к которому обращаются goroutines.
- Защитить доступ через mutex или передавать данные через channels.
- Проверить код с race detector.

## Как не повторить

- Не писать в общую map или slice из нескольких goroutines без защиты.
- Явно проектировать владельца данных.
- Использовать `context` и `WaitGroup` для управляемого завершения.
