# HTTP JSON Response

## Задача

Рецепт отправки JSON-ответа из HTTP-handler.

## Код

Код фиксируется после самостоятельной реализации. Минимальная идея: выставить `Content-Type`, выбрать HTTP-статус и закодировать структуру ответа в JSON.

## Объяснение

JSON-ответ должен быть единообразным во всем API. Это упрощает handlers и делает поведение API предсказуемым для клиента.

## Где использовать

- REST CRUD API.
- Ответы успешного создания, чтения и обновления ресурса.
- Ответы с ошибками, если используется единый envelope.

## Ограничения

- Заголовки нужно выставлять до записи тела ответа.
- Нельзя писать несколько статусов в один ответ.
- Ошибки кодирования JSON тоже нужно учитывать.
