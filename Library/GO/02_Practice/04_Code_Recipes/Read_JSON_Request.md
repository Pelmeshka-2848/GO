# Read JSON Request

## Задача

Чтение JSON-тела запроса.

## Код

Код фиксируется после самостоятельной реализации. В заметке важно сохранить не полный листинг, а идею: прочитать body, декодировать JSON в request-структуру, проверить ошибку декодирования и выполнить валидацию.

## Объяснение

Чтение JSON-запроса относится к transport-слою. Handler отвечает за формат HTTP-запроса, а не за бизнес-правила.

## Где использовать

- REST endpoints, которые принимают `POST`, `PUT` или `PATCH`.
- Лабораторные с JSON API.
- Сценарии, где нужно отделить ошибку формата от ошибки бизнес-логики.

## Ограничения

- Нельзя доверять данным после декодирования без валидации.
- Нужно ограничивать размер тела запроса в более серьезных приложениях.
- Ошибка JSON должна возвращаться как ошибка клиента.
