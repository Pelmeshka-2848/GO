# HTTP Error Response

## Задача

Единый формат HTTP-ошибок.

## Код

Код фиксируется после самостоятельной реализации. В заметке важно сохранить формат ошибки: HTTP-статус, короткое сообщение и при необходимости технический код ошибки.

## Объяснение

Единый формат ошибок помогает не размазывать обработку ошибок по handlers. Клиент получает предсказуемый JSON, а сервер сохраняет понятное разделение между ошибками клиента и ошибками сервера.

## Где использовать

- Невалидный JSON.
- Не найденный ресурс.
- Ошибка авторизации.
- Внутренняя ошибка сервера.

## Ограничения

- Не отдавать клиенту внутренние детали базы или stack trace.
- Не маскировать ошибки клиента статусом `500`.
- Не использовать один и тот же текст для разных классов ошибок.
