# Worker Pool

## Задача

Паттерн worker pool.

## Код

Код фиксируется после самостоятельной реализации. В заметке важно сохранить схему: очередь задач, фиксированное число workers, канал результатов и корректное завершение.

## Объяснение

Worker pool ограничивает параллелизм. Вместо запуска неограниченного числа goroutines задачи проходят через пул workers.

## Где использовать

- Проверка набора URL.
- Параллельная обработка файлов.
- Фоновые задачи с ограничением нагрузки.

## Ограничения

- Нужно закрывать каналы в правильном месте.
- Нельзя забывать про ожидание завершения workers.
- Для отмены работы лучше использовать context.
