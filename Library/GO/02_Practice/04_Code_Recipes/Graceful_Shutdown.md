# Graceful Shutdown

## Задача

Корректное завершение HTTP-сервера.

## Код

Код фиксируется после самостоятельной реализации. В заметке важно сохранить идею: слушать сигнал остановки, создать context с таймаутом и дать серверу завершить активные запросы.

## Объяснение

Graceful shutdown нужен, чтобы приложение завершалось управляемо: переставало принимать новые запросы, но давало текущим операциям короткое время на завершение.

## Где использовать

- HTTP-серверы.
- Production-ready API.
- Приложения с подключениями к базе, очередям или внешним сервисам.

## Ограничения

- Таймаут не должен быть бесконечным.
- Нужно закрывать дополнительные ресурсы, если они есть.
- В учебных проектах достаточно зафиксировать принцип и место подключения.
