# PostgreSQL Connection

## Задача

Подключение к PostgreSQL.

## Код

Код фиксируется после самостоятельной реализации. В заметке важно сохранить шаги: взять DSN из конфигурации, открыть подключение, проверить `Ping`, передать соединение в repository и закрыть ресурс при завершении.

## Объяснение

Подключение к PostgreSQL должно создаваться один раз при старте приложения. Handler не должен самостоятельно открывать соединение для каждого запроса.

## Где использовать

- REST API с PostgreSQL.
- CRUD-лабораторные.
- Проекты со слоем repository.

## Ограничения

- Не хранить реальные пароли в заметке.
- Не смешивать DSN, SQL-запросы и HTTP handlers.
- Проверять миграции отдельно от подключения.
