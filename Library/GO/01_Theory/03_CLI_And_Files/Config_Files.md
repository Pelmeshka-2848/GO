# Config Files

## Что это

Config file — файл с настройками приложения: порт, режим запуска, пути, параметры подключения, feature flags.

## Зачем нужно

Конфигурация отделяет поведение приложения от исходного кода. Один и тот же код можно запускать с разными настройками.

## Популярные форматы

- JSON;
- YAML;
- TOML;
- `.env`;
- обычные переменные окружения.

Для первых Go-лабораторных достаточно JSON или env-переменных.

## Что хранить в конфиге

- порт сервера;
- адрес базы данных;
- режим приложения;
- пути к файлам;
- лимиты и timeout-ы.

## Что не хранить в Git

- реальные пароли;
- production-токены;
- приватные ключи;
- персональные локальные настройки.

## Ключевые правила

- Храни пример конфига отдельно от реальных секретов.
- Указывай значения по умолчанию явно.
- Валидируй конфиг при старте приложения.
- Не читай конфиг из разных мест хаотично.
- Секреты лучше брать из env или secret storage, а не из обычного файла в репозитории.

## Частые ошибки

- закоммитить реальный пароль;
- менять код ради смены порта;
- не проверять обязательные настройки;
- смешивать config, constants и secrets.
