# File IO

## Что это

File I/O — это чтение и запись файлов: текстов, JSON, конфигов, логов, временных данных.

В Go для этого чаще всего используются пакеты `os`, `io`, `bufio` и иногда `path/filepath`.

## Зачем нужно

Работа с файлами нужна для CLI-утилит, конфигов, импорта данных, экспорта отчетов и локального хранения учебных результатов.

## Базовые операции

- прочитать файл целиком;
- открыть файл потоково;
- читать построчно;
- записать файл;
- создать директорию;
- проверить существование файла.

## Ключевые правила

- Всегда проверяй ошибки чтения и записи.
- Закрывай открытые файлы через `defer` после успешного открытия.
- Для путей используй `path/filepath`, если важна переносимость между ОС.
- Не перезаписывай файл без явного понимания последствий.
- Для больших файлов не читай все содержимое в память сразу.

## Частые ошибки

- вызвать `defer file.Close()` до проверки `err`;
- игнорировать ошибку записи;
- использовать строковую склейку путей вместо `filepath.Join`;
- читать большой файл целиком без необходимости;
- не учитывать кодировку и окончания строк.
