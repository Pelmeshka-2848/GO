# JSON

## Что это

JSON — текстовый формат обмена данными. В Go для работы с JSON используется стандартный пакет `encoding/json`.

## Зачем нужно

JSON встречается в REST API, конфигурационных файлах, логах, интеграциях и тестовых данных.

## Основные операции

- decode: JSON -> Go-структура;
- encode: Go-структура -> JSON;
- настройка имен полей через struct tags.

```go
type User struct {
    ID   int    `json:"id"`
    Name string `json:"name"`
}
```

## Экспорт полей

`encoding/json` работает только с экспортируемыми полями структуры. Поле `Name` попадет в JSON, поле `name` — нет.

## Ключевые правила

- Описывай JSON через структуры, а не через `map[string]any`, если формат известен.
- Проверяй ошибку декодирования.
- Для HTTP API возвращай единый формат ошибок.
- Используй JSON tags для стабильного внешнего контракта.
- Не доверяй входному JSON без валидации.

## Частые ошибки

- забыть экспортировать поля структуры;
- игнорировать ошибку `Decode`;
- не ограничить размер входного тела HTTP-запроса;
- смешать внутреннюю модель и внешний DTO без понимания последствий.
