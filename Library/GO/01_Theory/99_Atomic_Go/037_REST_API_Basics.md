# 037 — REST API Basics

## Кратко

REST API — это способ организовать HTTP-интерфейс вокруг ресурсов: задач, пользователей, заказов, файлов и других сущностей.

В учебном backend-проекте REST API помогает отделить:

- транспортный слой: HTTP-запросы и ответы;
- бизнес-логику: что приложение делает;
- хранение данных: память, файл или база данных.

## Основная идея

Ресурс получает URL, а действие над ресурсом выражается HTTP-методом.

| Действие | HTTP-метод | Пример пути |
|---|---|---|
| получить список | `GET` | `/tasks` |
| получить один объект | `GET` | `/tasks/{id}` |
| создать объект | `POST` | `/tasks` |
| обновить объект | `PUT` или `PATCH` | `/tasks/{id}` |
| удалить объект | `DELETE` | `/tasks/{id}` |

## Что важно в Go

- `net/http` дает базовый сервер и обработчики.
- Handler принимает `http.ResponseWriter` и `*http.Request`.
- JSON обычно кодируется и декодируется через `encoding/json`.
- Статус ответа нужно выставлять явно, если это не `200 OK`.
- Ошибки API лучше возвращать в едином формате.

## Минимальная модель мышления

1. Клиент отправляет HTTP-запрос.
2. Router или mux выбирает handler.
3. Handler читает входные данные.
4. Handler вызывает бизнес-логику.
5. Приложение возвращает статус и JSON-ответ.

## Частые ошибки

- смешивать HTTP-логику, бизнес-логику и SQL в одном большом handler;
- всегда возвращать `200 OK`, даже при ошибках;
- не проверять ошибки JSON-декодирования;
- не ограничивать список допустимых методов;
- хранить формат ответа разным в каждом endpoint-е.
