# Auth

## Что это

Auth обычно включает authentication и authorization.

Authentication отвечает на вопрос: кто пользователь. Authorization отвечает: что ему разрешено.

## Зачем нужно

Backend должен защищать приватные данные и действия: профиль, задачи, платежи, admin endpoint-ы.

## Типовой поток

1. Пользователь проходит login.
2. Сервер проверяет учетные данные.
3. Сервер выдает сессию или token.
4. Клиент отправляет token в защищенные запросы.
5. Сервер проверяет права.

## Правила

- Не хранить пароли в открытом виде.
- Разделять authentication и authorization.
- Проверять доступ на сервере, а не только на frontend.
- Не отдавать лишние детали при ошибке login.
- Логировать auth-события осторожно, без секретов.

## Частые ошибки

- считать JWT полной системой авторизации;
- хранить plain password;
- проверять права только в UI;
- возвращать “email существует” в public login flow без необходимости.
