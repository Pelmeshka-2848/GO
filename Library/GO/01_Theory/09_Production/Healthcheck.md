# Healthcheck

## Что это

Healthcheck — endpoint или проверка, которая сообщает, живо ли приложение и готово ли оно обслуживать запросы.

## Зачем нужно

Healthcheck используют orchestrator-ы, load balancer-ы, мониторинг и разработчики для быстрой проверки состояния сервиса.

## Виды

- liveness — процесс жив;
- readiness — сервис готов принимать запросы;
- dependency check — доступны ли БД/внешние зависимости.

## Правила

- Healthcheck должен быть быстрым.
- Liveness не должен зависеть от медленных внешних сервисов.
- Readiness может проверять критические зависимости.
- Не раскрывай чувствительную информацию в health response.
- Статус должен быть понятен для автоматических систем.

## Частые ошибки

- делать healthcheck слишком тяжелым;
- проверять все внешние API на каждый health request;
- отдавать stack trace или секреты;
- всегда возвращать OK, даже если БД недоступна.
