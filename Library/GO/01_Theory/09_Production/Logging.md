# Logging

## Что это

Logging — запись событий приложения: ошибок, запусков, запросов, важных изменений состояния.

## Зачем нужно

Логи помогают понять, что произошло в приложении без подключения debugger-а.

## Что логировать

- старт и остановку приложения;
- ошибки на границах системы;
- request ID и route;
- важные бизнес-события;
- проблемы интеграций;
- результат graceful shutdown.

## Что не логировать

- пароли;
- токены;
- полные connection strings;
- персональные данные без необходимости;
- огромные request/response body.

## Правила

- Лог должен иметь контекст.
- Ошибка логируется там, где она обработана или превращена в ответ.
- Для backend полезны структурированные логи.
- Log level должен соответствовать важности события.
- Не используй логи как замену нормальной обработки ошибок.

## Частые ошибки

- писать `error` без контекста;
- логировать одну ошибку на каждом слое;
- логировать секреты;
- использовать `fmt.Println` как постоянную систему логирования.
