# Observability

## Что это

Observability — способность понять состояние системы по внешним сигналам: логам, метрикам и трассировкам.

## Зачем нужно

Когда backend работает не локально, debugger уже не основной инструмент. Нужно видеть, что происходит в runtime.

## Три сигнала

- logs — что произошло;
- metrics — сколько, как часто, как долго;
- traces — путь запроса через компоненты.

## Что измерять

- latency endpoint-ов;
- error rate;
- количество запросов;
- состояние БД-пула;
- длину очередей;
- goroutines count;
- memory/CPU.

## Правила

- Добавляй request ID или trace ID.
- Метрики должны отвечать на эксплуатационные вопросы.
- Не логируй секреты.
- Ошибки внешних зависимостей должны быть видимы.
- Для production нужны alerts, а не только dashboards.

## Частые ошибки

- иметь логи без request ID;
- собирать метрики, которыми никто не пользуется;
- видеть ошибку, но не понимать, в каком запросе она случилась;
- не отличать user error от internal error.
