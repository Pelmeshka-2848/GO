# Config Env

## Что это

Config env — подход, при котором настройки приложения приходят из окружения: env-переменных, config-файлов, secret storage или параметров запуска.

## Зачем нужно

Один и тот же код должен запускаться локально, в тестах и production без переписывания исходников.

## Что обычно настраивается

- порт сервера;
- connection string базы данных;
- режим запуска;
- JWT secret или путь к ключам;
- log level;
- timeout-ы;
- адреса внешних API.

## Правила

- Обязательные настройки валидируются при старте.
- Секреты не имеют default-значений в коде.
- Значения по умолчанию допустимы для безопасных локальных настроек.
- Не логируй полный конфиг, если там есть секреты.
- Для примера можно хранить `.env.example`, но не реальный `.env`.

## Частые ошибки

- захардкодить production-пароль;
- менять порт через правку кода;
- не отличать пустую env-переменную от отсутствующей;
- запускать приложение с неполным конфигом и получать ошибку позже.
