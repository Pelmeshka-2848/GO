# Password Hashing

## Что это

Password hashing — преобразование пароля в безопасное одностороннее представление для хранения.

## Зачем нужно

Если база утечет, пароль не должен храниться в виде, пригодном для прямого использования.

## Что важно

Пароль не шифруют для хранения, а хэшируют алгоритмом, предназначенным для паролей.

## Подходящие алгоритмы

- bcrypt;
- Argon2;
- scrypt.

Обычные hash-функции вроде SHA-256 сами по себе не подходят для хранения паролей.

## Правила

- Храни только password hash.
- Используй соль и адаптивный алгоритм.
- Не логируй пароль.
- Не возвращай password hash в API.
- При проверке сравнивай введенный пароль с hash через библиотечную функцию.

## Частые ошибки

- хранить plain password;
- использовать быстрый SHA без password-specific алгоритма;
- отправлять hash клиенту;
- логировать входной password при ошибке.
