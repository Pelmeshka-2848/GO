# Go Naming Rules

## Правило

Имена в Go должны быть короткими, но не загадочными. Чем шире область видимости имени, тем понятнее оно должно быть.

## Почему так

Go-код часто читается сверху вниз небольшими функциями. Слишком длинные имена перегружают код, а слишком короткие скрывают смысл.

## Основные принципы

- Для локальных счетчиков допустимы короткие имена: `i`, `n`, `r`.
- Для значений с бизнес-смыслом нужны понятные имена: `userID`, `orderTotal`, `timeout`.
- Аббревиатуры обычно пишутся единообразно: `ID`, `HTTP`, `URL`.
- Экспортируемые имена начинаются с заглавной буквы.
- Неэкспортируемые имена начинаются со строчной буквы.
- Имя пакета должно быть коротким существительным: `http`, `user`, `storage`.

## Хорошо

```go
userID := 42
requestTimeout := 5 * time.Second
```

## Плохо

```go
x := 42
theUserIdentifierValue := 42
```

## Когда можно нарушить

- В маленьком цикле `i` читается лучше, чем `currentIndexOfUserCollection`.
- В математической формуле короткие имена могут соответствовать принятой записи.
- В тестах иногда допустимы компактные имена, если структура теста очевидна.
