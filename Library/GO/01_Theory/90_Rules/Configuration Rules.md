# Configuration Rules

## Правило

Конфигурация должна приходить извне приложения, а не быть зашита в коде.

## Почему так

Один и тот же бинарник должен запускаться в разных окружениях: local, test, staging, production. Отличаться должны настройки, а не исходный код.

## Хорошо

```text
APP_PORT=8080
DATABASE_URL=postgres://...
```

## Плохо

```go
const dbPassword = "production-password"
const port = "8080"
```

## Ключевые правила

- Секреты не хранятся в Git.
- У приложения должны быть безопасные значения по умолчанию.
- Обязательные настройки проверяются при старте.
- Ошибка конфигурации должна быть понятной.
- Пример `.env.example` можно хранить, реальный `.env` — нет.

## Когда можно нарушить

- В маленькой учебной программе без внешних зависимостей допустимы простые константы.
- Для тестов можно задавать настройки прямо в тестовом коде.
