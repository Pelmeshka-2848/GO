# Error Handling Rules

## Правило

Ошибки в Go нужно проверять явно и обрабатывать рядом с местом, где они возникли. Ошибку нельзя молча игнорировать.

## Почему так

В Go ошибка — обычное значение. Такой подход делает поток выполнения видимым: читатель кода сразу видит, где операция может не сработать и что программа делает в этом случае.

## Базовый паттерн

```go
value, err := doSomething()
if err != nil {
    return err
}
```

## Что делать с ошибкой

- вернуть выше, если текущий уровень не знает, как исправить проблему;
- добавить контекст, если без него ошибка непонятна;
- преобразовать в HTTP-ответ, если ошибка дошла до handler-а;
- залогировать на границе приложения, а не в каждом внутреннем слое.

## Плохо

```go
value, _ := doSomething()
_ = value
```

Такой код скрывает проблему и усложняет отладку.

## Частые ошибки

- использовать `_` для ошибки;
- логировать ошибку и одновременно возвращать ее выше без причины;
- возвращать пользователю внутренний текст SQL-ошибки;
- терять контекст: “failed” вместо “create user: email already exists”.

## Когда можно игнорировать ошибку

Редко. Только если операция гарантированно безопасна в данном контексте или ошибка реально не имеет значения. Даже тогда лучше оставить короткий комментарий.
