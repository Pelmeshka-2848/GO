# Worker Pool

## Что это

Worker pool — паттерн, где фиксированное количество workers обрабатывает поток задач из очереди.

## Зачем нужно

Worker pool ограничивает параллелизм. Это полезно, когда задач много, но запускать goroutine на каждую без лимита опасно.

## Модель

1. Producer кладет задачи в channel.
2. Несколько workers читают задачи.
3. Каждый worker обрабатывает задачу.
4. Результаты отправляются в отдельный channel или сохраняются синхронно.
5. После завершения задач channel закрывается.

## Где использовать

- проверка URL;
- обработка файлов;
- отправка запросов к внешнему API;
- фоновые jobs;
- ограничение нагрузки на базу данных.

## Ключевые правила

- Количество workers должно быть ограничено.
- Очередь задач должна закрываться после отправки всех jobs.
- Нужно понимать, где собираются ошибки.
- Context должен уметь остановить workers.
- Результаты должны собираться без data race.

## Частые ошибки

- не закрыть jobs channel;
- не дождаться завершения workers;
- писать результаты в общий slice без mutex;
- запускать слишком много workers;
- игнорировать отмену context.
