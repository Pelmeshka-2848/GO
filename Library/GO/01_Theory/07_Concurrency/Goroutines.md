# Goroutines

## Что это

Goroutine — легковесный поток выполнения, которым управляет runtime Go.

Запускается через ключевое слово `go`:

```go
go doWork()
```

## Зачем нужно

Goroutines позволяют выполнять несколько операций конкурентно: HTTP-запросы, обработку задач, чтение каналов, фоновые workers.

## Важная идея

Запуск goroutine не означает, что программа автоматически дождется ее завершения. Если `main` завершится, все goroutines процесса тоже завершатся.

## Ключевые правила

- Всегда понимай, кто ждет завершения goroutine.
- Всегда понимай, кто отменяет goroutine.
- Не запускай goroutine без контроля жизненного цикла.
- Для ожидания нескольких goroutines часто используется `sync.WaitGroup`.
- Для отмены долгих операций используй `context.Context`.

## Частые ошибки

- запустить goroutine и сразу завершить `main`;
- создать goroutine leak, которая никогда не завершается;
- писать в общую переменную без синхронизации;
- запускать неограниченное количество goroutines на входной поток данных.
