# Channels

## Что это

Channel — тип Go для передачи значений между goroutines.

```go
ch := make(chan int)
```

Через channel можно отправлять и получать значения одного типа.

## Зачем нужно

Channels помогают связывать конкурентные части программы без прямого доступа к общей памяти.

## Операции

Отправка:

```go
ch <- 10
```

Получение:

```go
value := <-ch
```

Закрытие:

```go
close(ch)
```

## Ключевые правила

- Отправитель обычно закрывает channel, а не получатель.
- Закрытие channel означает, что новых значений больше не будет.
- Чтение из закрытого channel возвращает нулевое значение и флаг `ok == false`.
- Не закрывай channel несколько раз.
- Не используй channel там, где простого mutex достаточно и понятнее.

## Частые ошибки

- deadlock из-за ожидания отправки или получения;
- закрыть channel со стороны получателя;
- отправить значение в закрытый channel;
- использовать channel как глобальный контейнер состояния;
- забыть, что unbuffered channel синхронизирует отправителя и получателя.
