# Buffered Channels

## Что это

Buffered channel — канал с ограниченной вместимостью.

```go
jobs := make(chan Job, 10)
```

Отправка блокируется только тогда, когда буфер заполнен. Чтение блокируется, когда буфер пуст.

## Зачем нужно

Buffered channels полезны для очередей задач, ограничения давления между producer и consumer, worker pool и временного накопления результатов.

## Важная идея

Буфер не делает программу “без блокировок”. Он только позволяет отправителю немного опережать получателя.

## Ключевые правила

- Размер буфера должен иметь смысл.
- Большой буфер может скрыть проблему производительности.
- Маленький буфер может усилить блокировки, но быстрее показывает backpressure.
- Для worker pool буфер часто равен числу задач или разумному лимиту очереди.

## Частые ошибки

- ставить огромный буфер вместо нормального управления нагрузкой;
- думать, что buffered channel гарантирует отсутствие deadlock;
- не закрыть channel после отправки всех задач;
- не обработать ситуацию, когда workers не успевают читать.
