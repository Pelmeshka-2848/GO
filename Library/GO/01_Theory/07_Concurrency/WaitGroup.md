# WaitGroup

## Что это

`sync.WaitGroup` позволяет дождаться завершения группы goroutines.

## Зачем нужно

Когда программа запускает несколько задач параллельно, ей часто нужно дождаться, пока все они закончатся.

## Основные операции

- `Add(n)` — добавить количество задач;
- `Done()` — отметить завершение одной задачи;
- `Wait()` — ждать, пока счетчик станет нулем.

## Типовой паттерн

```go
var wg sync.WaitGroup

wg.Add(1)
go func() {
    defer wg.Done()
    // work
}()

wg.Wait()
```

## Ключевые правила

- Вызывай `Add` до запуска goroutine.
- Внутри goroutine используй `defer wg.Done()`.
- Не копируй `WaitGroup` после использования.
- `WaitGroup` только ждет, но не собирает ошибки и не отменяет работу.
- Для ошибок и отмены добавляй channel, context или errgroup-подход.

## Частые ошибки

- вызвать `Add` внутри goroutine;
- забыть `Done`;
- вызвать `Done` больше раз, чем `Add`;
- ожидать, что `WaitGroup` сам остановит goroutines при ошибке.
