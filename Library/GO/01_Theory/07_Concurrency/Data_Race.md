# Data Race

## Что это

Data race возникает, когда несколько goroutines одновременно обращаются к одной памяти, хотя бы одна из них пишет, и доступ не синхронизирован.

## Почему это опасно

Результат становится непредсказуемым: программа может иногда работать, иногда ломаться, иногда возвращать неверные данные.

## Признаки риска

- общая переменная изменяется из нескольких goroutines;
- map читается и пишется конкурентно;
- счетчик увеличивается без mutex или atomic;
- shared slice меняется из нескольких worker-ов.

## Как искать

Go имеет race detector:

```bash
go test -race ./...
go run -race .
```

## Как исправлять

- использовать `sync.Mutex`;
- передавать данные через channel;
- ограничить владение состоянием одной goroutine;
- использовать atomic только для простых случаев и с пониманием.

## Частые ошибки

- думать, что race “редко проявляется, значит нормально”;
- защищать запись, но не защищать чтение;
- использовать map из нескольких goroutines без lock;
- передавать указатель на изменяемое состояние во все workers.
