# Unbuffered Channels

## Что это

Unbuffered channel — канал без буфера.

```go
ch := make(chan string)
```

Отправка в такой channel блокируется, пока другая goroutine не начнет чтение. Чтение блокируется, пока кто-то не отправит значение.

## Зачем нужно

Unbuffered channels полезны, когда нужна синхронизация между goroutines: значение передается только в момент, когда обе стороны готовы.

## Модель

Unbuffered channel — это не очередь. Это точка встречи отправителя и получателя.

## Ключевые правила

- Нужен активный отправитель и активный получатель.
- Если обе стороны ждут, программа может получить deadlock.
- Подходит для сигналов и строгой передачи результата.
- Для очереди задач чаще нужен buffered channel.

## Частые ошибки

- отправлять в unbuffered channel из той же goroutine без получателя;
- читать из channel, в который никто не пишет;
- использовать unbuffered channel как очередь;
- не закрывать channel, когда получатель ожидает завершения через `range`.
