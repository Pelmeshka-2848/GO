# Mutex

## Что это

`sync.Mutex` защищает общий ресурс от одновременного доступа несколькими goroutines.

## Зачем нужно

Mutex нужен, когда несколько goroutines читают и изменяют общую переменную, map, счетчик, cache или состояние объекта.

## Базовый паттерн

```go
mu.Lock()
defer mu.Unlock()

// work with shared state
```

## Ключевые правила

- Lock и Unlock должны быть парными.
- Чем меньше критическая секция, тем лучше.
- Не держи lock во время долгого I/O без необходимости.
- Для чтения и записи map из разных goroutines нужна синхронизация.
- Не копируй struct с mutex после использования.

## Частые ошибки

- забыть `Unlock`;
- держать lock слишком долго;
- читать map без lock, хотя запись идет под lock;
- получить deadlock из-за повторного lock тем же потоком выполнения;
- копировать mutex вместе со структурой.
