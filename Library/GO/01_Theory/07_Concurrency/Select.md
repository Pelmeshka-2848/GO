# Select

## Что это

`select` позволяет ждать несколько channel-операций одновременно.

```go
select {
case value := <-resultCh:
    _ = value
case <-ctx.Done():
    return ctx.Err()
}
```

## Зачем нужно

`select` нужен для конкурентного ожидания: результат, отмена, timeout, сигнал остановки, несколько источников данных.

## Поведение

- Если готов один case, выполняется он.
- Если готовы несколько case, Go выбирает один псевдослучайно.
- Если ни один case не готов, `select` блокируется.
- `default` делает `select` неблокирующим.

## Ключевые правила

- Используй `select` для отмены через `ctx.Done()`.
- Будь осторожен с `default`: он может создать busy loop.
- Помни, что порядок case не задает приоритет.
- Для timeout часто используют context или timer.

## Частые ошибки

- ожидать, что первый case имеет приоритет;
- добавить `default` и получить цикл, нагружающий CPU;
- не обрабатывать закрытие channel;
- забыть case отмены и получить зависшую goroutine.
