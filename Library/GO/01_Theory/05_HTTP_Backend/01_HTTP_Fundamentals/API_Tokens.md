# API Tokens

## Что это

API token — секретное значение, которое подтверждает право приложения обращаться к API.

## Зачем нужно

Токены используются для доступа к внешним сервисам, внутренним API, webhooks, bot API и admin endpoint-ам.

## Где хранить

- переменные окружения;
- secret manager;
- локальный `.env`, если он не попадает в Git;
- настройки CI/CD.

## Где не хранить

- в исходном коде;
- в публичном README;
- в Obsidian-заметках с реальным значением;
- в логах.

## Ключевые правила

- Не коммить реальные токены.
- Используй разные токены для dev и production.
- Ротируй токен, если он утек.
- Логируй только маскированное значение, если это вообще нужно.
- Добавляй `.env` в `.gitignore`.

## Частые ошибки

- вставить токен прямо в код;
- отправить токен в лог;
- использовать один токен везде;
- не удалить токен после публикации репозитория.
