# Handlers

## Что это

Handler — функция или объект, который обрабатывает HTTP-запрос.

В стандартном `net/http` handler обычно получает `http.ResponseWriter` и `*http.Request`.

## Зачем нужно

Handler является границей между HTTP-миром и логикой приложения. Он читает запрос, вызывает нужную логику и формирует ответ.

## Ответственность handler-а

- проверить HTTP-метод;
- прочитать path, query, headers или body;
- вызвать сервисный слой;
- выбрать HTTP-статус;
- записать ответ.

## Что не стоит держать в handler-е

- сложную бизнес-логику;
- SQL-запросы;
- большую валидацию без выделения функций;
- повторяющийся код формирования ошибок.

## Ключевые правила

- Handler должен быть тонким.
- Ошибки приложения нужно преобразовывать в HTTP-ответы на границе handler-а.
- Повторяющуюся логику выноси в middleware или helper-функции.
- Не забывай выставлять статус до записи тела ответа.

## Частые ошибки

- писать в `ResponseWriter` до выбора статуса;
- смешивать JSON-декодирование, SQL и бизнес-правила в одной функции;
- не ограничивать HTTP-метод;
- возвращать разные форматы ошибок в разных handler-ах.
