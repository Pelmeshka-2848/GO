# Rate Limits

## Что это

Rate limit — ограничение количества запросов за период времени.

## Зачем нужно

Ограничения защищают сервисы от перегрузки, злоупотреблений и случайных циклов запросов.

## Где встречается

- внешние API;
- auth endpoint-ы;
- публичные REST API;
- Telegram Bot API;
- микросервисные вызовы.

## Как реагировать

- проверять HTTP-статус `429 Too Many Requests`;
- учитывать заголовки лимитов, если API их возвращает;
- использовать backoff перед повтором;
- не делать бесконечный retry;
- логировать превышение лимита.

## Ключевые правила

- Уважай лимиты внешнего API.
- Для своих публичных endpoint-ов добавляй защиту на чувствительные маршруты.
- Retry должен иметь ограничение.
- Rate limiting лучше выносить в middleware или инфраструктурный слой.

## Частые ошибки

- игнорировать `429`;
- повторять запросы без задержки;
- делать retry для неидемпотентной операции без понимания последствий;
- не отличать лимит клиента от падения сервиса.
