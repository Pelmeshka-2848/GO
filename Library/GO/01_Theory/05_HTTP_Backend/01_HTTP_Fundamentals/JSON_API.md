# JSON API

## Что это

JSON API — HTTP API, где запросы и ответы передаются в формате JSON.

## Зачем нужно

JSON прост, читаем человеком и поддерживается почти всеми клиентами: браузером, мобильными приложениями, backend-сервисами и CLI.

## Типовой ответ

```json
{
  "id": 1,
  "title": "Learn Go"
}
```

## Типовая ошибка

```json
{
  "error": "invalid request"
}
```

## Ключевые правила

- Используй `Content-Type: application/json`.
- Описывай внешний контракт через DTO-структуры.
- Проверяй ошибки JSON-декодирования.
- Возвращай единый формат ошибок.
- Не отдавай внутренние поля, которые клиенту не нужны.

## Частые ошибки

- забыть экспортировать поля структуры;
- игнорировать ошибку декодирования body;
- смешать internal model и API response;
- возвращать разные форматы ответа в разных endpoint-ах.
