# HTTP Status Codes

## Что это

HTTP status code — числовой код результата обработки запроса.

## Основные группы

| Группа | Смысл | Примеры |
|---|---|---|
| `2xx` | успех | `200`, `201`, `204` |
| `3xx` | перенаправление | `301`, `302` |
| `4xx` | ошибка клиента | `400`, `401`, `403`, `404`, `409` |
| `5xx` | ошибка сервера | `500`, `503` |

## Частые коды API

- `200 OK` — успешный ответ;
- `201 Created` — ресурс создан;
- `204 No Content` — успешно, тела ответа нет;
- `400 Bad Request` — неверный запрос;
- `401 Unauthorized` — нет или неверна аутентификация;
- `403 Forbidden` — доступ запрещен;
- `404 Not Found` — ресурс не найден;
- `409 Conflict` — конфликт состояния;
- `500 Internal Server Error` — внутренняя ошибка сервера.

## Ключевые правила

- Статус должен отражать смысл результата.
- Не возвращай `200 OK` для ошибок.
- Для создания ресурса обычно подходит `201 Created`.
- Для удаления без тела ответа подходит `204 No Content`.
- Внутренние ошибки не должны раскрывать детали реализации.

## Частые ошибки

- путать `401` и `403`;
- использовать `500` для ошибок валидации;
- отправить body после `204`;
- записать body до установки статуса.
