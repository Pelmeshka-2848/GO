# REST API

## Что это

REST API организует HTTP-интерфейс вокруг ресурсов. Ресурс имеет URL, а действие выражается HTTP-методом.

## Пример ресурса

Для ресурса `tasks` типовые endpoint-ы выглядят так:

| Метод | Путь | Смысл |
|---|---|---|
| `GET` | `/tasks` | список задач |
| `GET` | `/tasks/{id}` | одна задача |
| `POST` | `/tasks` | создать задачу |
| `PUT` | `/tasks/{id}` | заменить задачу |
| `PATCH` | `/tasks/{id}` | частично обновить задачу |
| `DELETE` | `/tasks/{id}` | удалить задачу |

## Ключевые правила

- URL должен описывать ресурс, а не действие.
- Действие выбирается HTTP-методом.
- Статусы ответа должны соответствовать результату.
- Ошибки лучше возвращать в едином формате.
- Handler не должен содержать всю бизнес-логику.

## Частые ошибки

- делать пути вроде `/getTasks` и `/deleteTask`;
- возвращать `200 OK` для всех случаев;
- смешивать REST API и внутреннюю структуру БД;
- не версионировать API, если контракт уже используется клиентами.
