# External API

## Что это

External API — API внешнего сервиса, к которому обращается приложение.

## Зачем нужно

Backend часто интегрируется с внешними системами: платежи, уведомления, карты, Telegram, OAuth, почта.

## Что учитывать

- авторизацию;
- формат запроса;
- формат ответа;
- лимиты запросов;
- timeout;
- retry;
- ошибки внешнего сервиса;
- изменение контракта API.

## Ключевые правила

- Изолируй внешний API за отдельным клиентом или адаптером.
- Не размазывай URL и токены по handler-ам.
- Логируй контекст ошибки, но не секреты.
- Обрабатывай rate limit и временную недоступность.
- Добавляй timeout на каждый внешний вызов.

## Частые ошибки

- вызывать внешний API прямо из handler-а без слоя клиента;
- хранить токен в коде;
- не проверять status code;
- бесконечно повторять запрос при ошибке;
- не иметь тестовой fake-реализации клиента.
