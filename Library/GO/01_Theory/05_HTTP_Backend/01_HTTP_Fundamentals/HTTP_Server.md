# HTTP Server

## Что это

HTTP-сервер принимает HTTP-запросы, передает их обработчикам и возвращает HTTP-ответы клиенту.

В Go базовый HTTP-сервер доступен в стандартном пакете `net/http`.

## Зачем нужно

HTTP-сервер — основа большинства backend-приложений: REST API, webhooks, healthcheck endpoint-ы, внутренние сервисы.

## Минимальная модель

1. Приложение регистрирует маршруты.
2. Сервер слушает порт.
3. Клиент отправляет запрос.
4. Handler обрабатывает запрос.
5. Сервер возвращает ответ.

## Ключевые правила

- Не держи всю бизнес-логику внутри `main`.
- Проверяй ошибку запуска сервера.
- Явно выбирай порт и фиксируй его в конфигурации.
- Для production-сервера нужны timeout-ы.
- Health endpoint помогает проверять, что сервис жив.

## Частые ошибки

- игнорировать ошибку `ListenAndServe`;
- запускать сервер на занятом порту;
- не разделять маршрутизацию, handler-ы и бизнес-логику;
- не задавать timeout-ы на сервере.
