# Microservices

## Что это

Microservices — архитектурный стиль, где система состоит из нескольких независимых сервисов, каждый отвечает за свою область.

## Зачем нужно

Микросервисы помогают независимо развивать, масштабировать и разворачивать части большой системы.

## Цена подхода

Микросервисы добавляют сложность:

- сеть между сервисами;
- distributed tracing;
- retries и timeouts;
- eventual consistency;
- service discovery;
- отдельный deploy;
- мониторинг каждого сервиса.

## Правила

- Не начинать с микросервисов без причины.
- Граница сервиса должна идти по бизнес-возможности, а не по техническому слою.
- У каждого сервиса должна быть своя зона ответственности.
- Межсервисные вызовы требуют timeout, retry policy и observability.

## Типичные ошибки

- разделить один CRUD на несколько сервисов слишком рано;
- сделать “распределенный монолит”;
- использовать общую базу данных без ясных правил;
- забыть про tracing и логи.
