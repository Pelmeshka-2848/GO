# Cache

## Что это

Cache — временное хранилище данных для ускорения доступа или снижения нагрузки на источник.

## Зачем нужно

Кэш помогает не ходить каждый раз в базу или внешний API, если данные можно безопасно переиспользовать.

## Виды кэша

- in-memory cache;
- distributed cache;
- HTTP cache;
- database query cache;
- CDN cache.

## Правила

- У кэша должна быть стратегия инвалидирования.
- Не кэшируй чувствительные данные без понимания рисков.
- Cache miss должен корректно обрабатываться.
- Кэш не должен быть единственным источником истины, если это не отдельное архитектурное решение.
- Нужны метрики hit/miss для production.

## Типичные ошибки

- кэшировать всё подряд;
- не очищать устаревшие данные;
- забыть про race condition в in-memory cache;
- сделать кэш частью бизнес-истины случайно.
