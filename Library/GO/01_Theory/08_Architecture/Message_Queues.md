# Message Queues

## Что это

Message queue — инфраструктурный компонент для асинхронной передачи задач или событий между частями системы.

## Зачем нужно

Очереди помогают разгрузить request path, повысить устойчивость и обрабатывать фоновые задачи отдельно.

## Примеры задач

- отправка email;
- обработка изображений;
- интеграция с внешним API;
- пересчет статистики;
- обмен событиями между сервисами.

## Правила

- Сообщение должно иметь понятный контракт.
- Consumer должен быть идемпотентным, если возможны повторы.
- Нужна стратегия retry и dead-letter queue.
- Очередь не должна скрывать ошибки навсегда.
- Важны метрики длины очереди и ошибок обработки.

## Типичные ошибки

- считать очередь заменой нормальной модели данных;
- не обрабатывать повторную доставку;
- хранить слишком много бизнес-логики в consumer без тестов;
- не иметь visibility по ошибкам.
