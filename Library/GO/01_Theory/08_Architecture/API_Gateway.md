# API Gateway

## Что это

API Gateway — входная точка перед несколькими backend-сервисами.

## Зачем нужно

Gateway может централизовать routing, auth, rate limiting, CORS, TLS termination и агрегацию ответов.

## Что может делать

- маршрутизировать запросы;
- проверять токены;
- ограничивать запросы;
- скрывать внутреннюю структуру сервисов;
- агрегировать данные из нескольких сервисов.

## Что не должен делать

- превращаться в огромный монолит бизнес-логики;
- знать все детали домена каждого сервиса;
- быть единственной точкой, где живет безопасность.

## Правила

- Gateway должен быть тонким.
- Бизнес-правила остаются в сервисах.
- Нужны timeouts и circuit-breaking подходы для downstream.
- Логи и tracing особенно важны.

## Типичные ошибки

- перенести всю логику в gateway;
- сделать gateway single point of failure без отказоустойчивости;
- не настроить timeout к downstream-сервисам;
- скрыть ошибки так, что сервисы невозможно отлаживать.
