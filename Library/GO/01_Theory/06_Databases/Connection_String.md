# Connection String

## Что это

Connection string — строка подключения к базе данных. Она содержит адрес, порт, имя базы, пользователя, пароль и дополнительные параметры.

## Пример формы

```text
postgres://user:password@localhost:5432/app_db?sslmode=disable
```

Это пример формата, а не значение для хранения в публичном репозитории.

## Зачем нужно

Приложение использует connection string, чтобы driver понял, куда подключаться и с какими учетными данными.

## Что обычно входит

- host;
- port;
- database name;
- username;
- password;
- SSL mode;
- connection options.

## Ключевые правила

- Реальные пароли не хранятся в коде.
- Для локальной разработки можно использовать `.env`, не попадающий в Git.
- Для production используются переменные окружения или secret storage.
- Ошибка подключения должна быть понятной в логах, но без утечки пароля.

## Частые ошибки

- закоммитить пароль;
- забыть `sslmode` для локальной разработки;
- подключаться не к той базе;
- путать имя контейнера Docker и `localhost`;
- логировать полную строку подключения.
