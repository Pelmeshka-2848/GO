# PostgreSQL

## Что это

PostgreSQL — реляционная база данных с поддержкой SQL, транзакций, индексов, ограничений целостности и расширений.

## Зачем нужно

Backend-приложение использует PostgreSQL для надежного хранения данных: пользователей, задач, заказов, сессий, событий и связей между сущностями.

## Базовые сущности

- database — база данных;
- schema — пространство имен объектов;
- table — таблица;
- row — строка;
- column — колонка;
- index — структура для ускорения поиска;
- constraint — ограничение целостности.

## Что важно для Go

- Go-приложение подключается к PostgreSQL через driver или pgx.
- Запросы должны выполняться с context.
- Ошибки базы нужно обрабатывать и не отдавать пользователю напрямую.
- Схема БД должна управляться миграциями.

## Ключевые правила

- Описывай структуру данных через таблицы и связи.
- Используй primary key.
- Добавляй constraints для важных правил данных.
- Не храни пароль к БД в коде.
- Не собирай SQL из пользовательского ввода конкатенацией строк.

## Частые ошибки

- держать схему БД только “в голове”;
- менять таблицы вручную без миграций;
- не закрывать rows после запроса;
- не отличать ошибку подключения от ошибки SQL-запроса;
- хранить connection string в репозитории с реальным паролем.
