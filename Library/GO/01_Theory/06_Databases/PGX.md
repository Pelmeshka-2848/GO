# PGX

## Что это

`pgx` — популярный Go-драйвер и toolkit для работы с PostgreSQL.

## Зачем нужно

`pgx` дает хороший PostgreSQL-native опыт: пул подключений, работу с типами PostgreSQL, batch-запросы и удобные API.

## Два режима использования

- через совместимость с `database/sql`;
- напрямую через `pgx` и `pgxpool`.

Прямой режим часто выбирают, когда проект точно использует PostgreSQL и не нужен абстрактный SQL-интерфейс.

## Ключевые правила

- Используй `pgxpool.Pool` как общий пул подключений.
- Передавай context в запросы.
- Закрывай pool при завершении приложения.
- Не прячь реальные ошибки PostgreSQL там, где они важны для бизнес-логики.
- Изолируй работу с pgx в repository layer.

## Частые ошибки

- создавать pool на каждый запрос;
- забыть закрыть pool;
- не обрабатывать ошибку подключения при старте;
- привязать весь проект к pgx-типам без необходимости;
- смешать SQL и HTTP в одном handler-е.
