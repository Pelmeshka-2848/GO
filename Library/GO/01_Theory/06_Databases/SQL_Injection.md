# SQL Injection

## Что это

SQL injection — уязвимость, при которой пользовательский ввод становится частью SQL-команды и меняет ее смысл.

## Почему опасно

Атакующий может прочитать чужие данные, изменить записи, обойти авторизацию или повредить базу.

## Плохая идея

```go
query := "SELECT * FROM users WHERE email = '" + email + "'"
```

Так пользовательское значение напрямую попадает в SQL.

## Правильная идея

Использовать параметры запроса:

```go
SELECT * FROM users WHERE email = $1
```

Значение передается отдельно от текста SQL, и база не воспринимает его как команду.

## Ключевые правила

- Не собирай SQL через конкатенацию пользовательского ввода.
- Используй placeholders и параметры.
- Валидируй входные данные, но не считай валидацию заменой параметров.
- Осторожно работай с динамическими ORDER BY и именами колонок.
- Логи не должны раскрывать чувствительные данные.

## Частые ошибки

- параметризовать значения, но склеивать имя колонки из запроса;
- считать, что escaping строк решает проблему полностью;
- строить фильтры через конкатенацию;
- использовать пользовательский ввод в `LIMIT`, `ORDER BY`, `WHERE` без whitelist.
