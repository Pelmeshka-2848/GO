# database/sql

## Что это

`database/sql` — стандартный пакет Go, который задает общий интерфейс для работы с SQL-базами через драйверы.

## Зачем нужно

Пакет позволяет выполнять SQL-запросы, управлять подключениями, транзакциями и читать результаты.

## Основные типы

- `sql.DB` — пул подключений, а не одно соединение;
- `sql.Row` — один результат;
- `sql.Rows` — набор строк;
- `sql.Tx` — транзакция;
- `sql.Stmt` — подготовленный statement.

## Важная идея

`sql.DB` нужно создавать один раз на приложение и переиспользовать. Не нужно открывать новое подключение на каждый запрос.

## Ключевые правила

- Передавай `context.Context` в запросы через методы `QueryContext`, `ExecContext`, `QueryRowContext`.
- Закрывай `Rows` через `defer rows.Close()`.
- Проверяй `rows.Err()` после обхода.
- Различай `sql.ErrNoRows` и другие ошибки.
- Настраивай пул соединений осознанно.

## Частые ошибки

- считать `sql.DB` одним физическим соединением;
- открывать `sql.DB` на каждый HTTP-запрос;
- забыть `rows.Close`;
- не проверить ошибку после `Scan`;
- смешать SQL-логику с handler-ами.
