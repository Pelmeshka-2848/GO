# Transactions

## Что это

Транзакция — группа операций с базой данных, которая должна выполниться целиком или не выполниться вообще.

## Зачем нужно

Транзакции защищают целостность данных. Например, нельзя создать заказ без связанных позиций или списать деньги без записи операции.

## ACID кратко

- Atomicity — все или ничего;
- Consistency — данные остаются корректными;
- Isolation — параллельные операции не ломают друг друга;
- Durability — после commit данные сохранены.

## Типовой сценарий

1. Begin.
2. Выполнить несколько запросов.
3. Если все успешно — Commit.
4. Если ошибка — Rollback.

## Ключевые правила

- Всегда обрабатывай ошибку `Commit`.
- При ошибке делай rollback.
- Не держи транзакцию открытой дольше нужного.
- Не выполняй внешние HTTP-вызовы внутри транзакции без крайней необходимости.
- Передавай транзакцию явно в repository-операции, которые должны быть атомарными.

## Частые ошибки

- забыть rollback;
- держать транзакцию во время долгой сетевой операции;
- смешать запросы вне транзакции и внутри нее;
- думать, что транзакция автоматически решает все race conditions бизнес-логики.
