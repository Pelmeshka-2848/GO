# Table Driven Tests

## Что это

Table-driven test — тест, где набор входных данных и ожидаемых результатов описан таблицей cases.

## Зачем нужно

Такой стиль удобен, когда одна функция или сценарий проверяется на множестве похожих случаев.

## Модель

```go
tests := []struct {
    name string
    input string
    want bool
}{
    {name: "empty", input: "", want: false},
}
```

Это форма, а не готовое решение конкретной лабораторной.

## Правила

- У каждого case должно быть имя.
- Проверяй и успешные, и ошибочные сценарии.
- Используй `t.Run`, чтобы видеть, какой case упал.
- Не делай таблицу слишком большой и нечитаемой.
- Если setup сильно разный, возможно, нужны отдельные тесты.

## Типичные ошибки

- cases без имени;
- слишком много полей в таблице;
- один тест проверяет несколько разных поведений;
- забыть проверить ошибку отдельно от результата.
