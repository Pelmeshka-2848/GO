# Benchmarks

## Что это

Benchmark — тест производительности, который измеряет скорость выполнения кода.

## Зачем нужно

Benchmarks помогают сравнивать реализации и находить дорогие операции, но только после появления реальной потребности.

## Форма benchmark-а

```go
func BenchmarkSomething(b *testing.B) {
    for i := 0; i < b.N; i++ {
        // operation
    }
}
```

## Запуск

```bash
go test -bench=. ./...
```

## Правила

- Сначала корректность, потом производительность.
- Benchmark должен измерять именно нужную операцию.
- Setup не должен попадать в измерение, если он не часть операции.
- Сравнивай результаты на одинаковой машине и условиях.
- Для памяти используй `-benchmem`.

## Типичные ошибки

- оптимизировать без bottleneck-а;
- измерять setup вместо операции;
- делать вывод по одному запуску;
- забыть, что compiler может оптимизировать неиспользуемый результат.
