# Error Wrapping

## Что это

Error wrapping — добавление контекста к ошибке без потери исходной причины.

## Зачем нужно

Ошибка `not found` сама по себе мало говорит. Ошибка `get user: query database: not found` помогает понять путь сбоя.

## Основной инструмент

```go
return fmt.Errorf("read config: %w", err)
```

`%w` сохраняет исходную ошибку для `errors.Is` и `errors.As`.

## Когда использовать

- при переходе через слой;
- при добавлении имени операции;
- перед возвратом ошибки наверх;
- при сохранении технической причины для логов.

## Правила

- Добавляй полезный контекст, а не шум.
- Для проверки wrapped-ошибок используй `errors.Is` или `errors.As`.
- Не сравнивай текст ошибки.
- Не логируй и не wrap-ь одну ошибку бессмысленно на каждом уровне.

## Типичные ошибки

- использовать `%v` вместо `%w`;
- потерять sentinel error;
- вернуть пользователю внутреннюю wrapped-цепочку;
- добавить контекст без указания операции.
