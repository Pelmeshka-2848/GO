# Testing

## Что это

Testing в Go — автоматическая проверка поведения кода через файлы `_test.go` и пакет `testing`.

## Зачем нужно

Тесты фиксируют ожидаемое поведение и помогают менять код без страха сломать уже работающие сценарии.

## Что тестировать

- чистые функции;
- бизнес-правила;
- обработку ошибок;
- HTTP handlers;
- repository-логику с тестовой БД;
- граничные случаи.

## Правила

- Тест должен быть воспроизводимым.
- Название теста должно описывать поведение.
- Проверяй happy path и error path.
- Не завязывай unit-тест на внешние сервисы.
- Для похожих cases используй table-driven tests.

## Типичные ошибки

- тестировать только успешный сценарий;
- писать тест, который зависит от времени или порядка выполнения;
- проверять внутренние шаги вместо результата;
- требовать реальную production-БД для unit-теста.
