# Reflection

## Что это

Reflection — возможность анализировать типы и значения во время выполнения через пакет `reflect`.

## Зачем нужно

Reflection используется в библиотеках сериализации, ORM, DI-контейнерах, validators и generic utilities.

## Почему осторожно

Reflection снижает читаемость, может быть медленнее обычного кода и переносит часть ошибок из compile-time в runtime.

## Когда оправдано

- библиотечный код;
- работа с tags;
- универсальная сериализация;
- tooling;
- редкие инфраструктурные helper-ы.

## Когда не нужно

- обычная бизнес-логика;
- CRUD handler;
- простое преобразование struct;
- случай “не хочу писать явный код”.

## Типичные ошибки

- использовать reflection вместо интерфейса;
- паниковать из-за неверного kind/value;
- скрывать простую логику за магией;
- писать runtime-код, который компилятор мог бы проверить.
