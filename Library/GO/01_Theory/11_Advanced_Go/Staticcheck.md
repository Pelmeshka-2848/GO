# Staticcheck

## Что это

Staticcheck — сторонний статический анализатор Go-кода, который находит bugs, dead code, style-проблемы и неэффективные конструкции.

## Зачем нужно

Он дополняет compiler и `go vet`, помогая поддерживать качество кода в проекте.

## Что может находить

- неиспользуемую или подозрительную логику;
- неправильную работу с errors;
- unreachable code;
- deprecated API;
- неэффективные конструкции;
- concurrency mistakes.

## Правила

- Staticcheck полезен в CI.
- Перед массовым исправлением предупреждений нужно понять их смысл.
- Не все замечания одинаково критичны.
- Для учебного проекта можно запускать периодически, а не после каждого шага.

## Типичные ошибки

- воспринимать staticcheck как форматтер;
- слепо исправлять без понимания;
- игнорировать реальные bugs;
- не фиксировать версию инструмента в командной среде проекта.
