# Go Build

## Что это

`go build` компилирует Go-проект и создает исполняемый файл для `package main`.

```bash
go build
go build -o app .
```

## Зачем нужно

`go build` нужен, чтобы проверить компиляцию проекта и получить бинарник для запуска без `go run`.

## Основные варианты

```bash
go build .
go build ./...
go build -o bin/app ./cmd/app
```

`go build ./...` проверяет сборку всех пакетов в модуле.

## Ключевые правила

- Перед коммитом полезно проверить `go build ./...`.
- Бинарники не стоит хранить в Git.
- Имя выходного файла задается через `-o`.
- Сборка библиотеки проверяет компиляцию, но не создает исполняемый файл.

## Частые ошибки

- ждать бинарник от пакета, который не является `package main`;
- добавить собранный `.exe` или binary в репозиторий;
- забыть пересобрать проект после изменения зависимостей;
- не отличать ошибку сборки от ошибки тестов.
