# Go Installation

## Что это

Установка Go — это подготовка локального инструментария: компилятора, стандартной библиотеки и команды `go`.

## Зачем нужно

Без установленного Go нельзя запускать, собирать, тестировать и форматировать Go-проекты.

## Проверка установки

```bash
go version
go env GOPATH
go env GOROOT
```

`GOROOT` указывает на установленный Go SDK. `GOPATH` используется для кэшей, скачанных модулей и старого workspace-подхода.

## Что должно работать

- команда `go version` показывает версию;
- `go env` выводит настройки окружения;
- IDE видит Go SDK;
- простой файл `main.go` запускается через `go run`.

## Ключевые правила

- Ставить Go лучше из официального установщика или через нормальный пакетный менеджер.
- После установки проверь `PATH`.
- Не смешивай несколько установок Go без понимания, какая используется.
- Для проекта важна версия Go в `go.mod`.

## Частые ошибки

- `go` не найден в терминале из-за неправильного `PATH`;
- IDE настроена на другой SDK, чем терминал;
- установлен старый Go, а проект требует более новую версию;
- путаница между `GOROOT` и `GOPATH`.
