# Go Test

## Что это

`go test` запускает тесты Go-пакетов.

```bash
go test ./...
```

## Зачем нужно

Тесты помогают проверять поведение кода автоматически: функции, сервисы, handlers, работу с ошибками и граничные случаи.

## Основные команды

```bash
go test .
go test ./...
go test -v ./...
go test -run TestName ./...
```

## Файлы тестов

Тесты лежат в файлах с суффиксом `_test.go`. Тестовая функция обычно начинается с `Test`.

```go
func TestAdd(t *testing.T) {
    // учебный фрагмент формы теста
}
```

## Ключевые правила

- Запускай `go test ./...` перед фиксацией изменений.
- Тестируй поведение, а не внутреннюю реализацию.
- Для набора похожих случаев используй table-driven tests.
- Ошибочные сценарии важны не меньше успешных.

## Частые ошибки

- тестировать только happy path;
- забыть `./...` и проверить только текущий пакет;
- писать тест, который зависит от порядка выполнения;
- смешивать unit-тест и интеграционный тест без явного разделения.
