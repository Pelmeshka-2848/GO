# Go Mod Tidy

## Что это

`go mod tidy` приводит `go.mod` и `go.sum` в соответствие с реальными import-ами проекта.

```bash
go mod tidy
```

## Зачем нужно

Команда удаляет неиспользуемые зависимости и добавляет недостающие записи для используемых пакетов.

## Когда запускать

- после добавления нового import-а из внешнего модуля;
- после удаления кода, который использовал зависимость;
- перед коммитом;
- после обновления версии Go или зависимостей.

## Ключевые правила

- Запускай команду из корня модуля.
- Коммить изменения `go.mod` и `go.sum` вместе с кодом.
- Не правь `go.sum` вручную.
- Если `tidy` удаляет зависимость, значит проект ее реально не импортирует.

## Частые ошибки

- запускать `go mod tidy` вне папки с `go.mod`;
- удивляться, что зависимость исчезла из `go.mod`;
- не коммитить обновленный `go.sum`;
- использовать `go get` вместо `tidy` для очистки зависимостей.
