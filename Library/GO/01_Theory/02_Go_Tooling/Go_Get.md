# Go Get

## Что это

`go get` добавляет или обновляет зависимости Go-модуля.

```bash
go get github.com/example/lib
go get github.com/example/lib@v1.2.3
```

## Зачем нужно

Команда нужна, когда проект использует внешний пакет: router, драйвер базы данных, библиотеку для JWT, миграции и т.д.

## Версии

Можно указать конкретную версию:

```bash
go get example.com/module@v1.4.0
```

Можно обновить до последней подходящей:

```bash
go get -u example.com/module
```

## Ключевые правила

- После `go get` проверь изменения в `go.mod` и `go.sum`.
- Для учебного проекта фиксируй, зачем добавлена зависимость.
- Не добавляй библиотеку, если задача решается стандартной библиотекой.
- Для инструментов разработки иногда лучше использовать отдельную схему фиксации tool dependencies.

## Частые ошибки

- добавлять тяжелую зависимость без необходимости;
- не понимать, какая версия установилась;
- обновить много зависимостей сразу и получить неожиданные изменения;
- забыть запустить тесты после обновления.
