# Go Modules

## Что это

Go module — это единица управления проектом и зависимостями. Модуль описывается файлом `go.mod`.

```bash
go mod init example/project
```

## Зачем нужно

Модули позволяют:

- задать имя модуля;
- указать версию Go;
- фиксировать зависимости;
- воспроизводимо собирать проект на другой машине.

## Файл go.mod

Минимальный `go.mod` содержит имя модуля и версию Go:

```text
module example/project

go 1.22
```

Имя модуля используется в import-путях внутри проекта и при публикации.

## Ключевые правила

- Один проект обычно имеет один `go.mod` в корне.
- Имя модуля должно быть стабильным.
- Для локального учебного проекта допустимо имя вроде `example/lab-name`.
- Не редактируй `go.sum` вручную.
- После изменения зависимостей запускай `go mod tidy`.

## Частые ошибки

- создать `go.mod` не в той папке;
- выбрать случайное имя модуля, а потом путаться в import-ах;
- удалить `go.sum`, не понимая зачем он нужен;
- коммитить зависимости как папки vendor без необходимости.
