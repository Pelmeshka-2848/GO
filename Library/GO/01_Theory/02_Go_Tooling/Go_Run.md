# Go Run

## Что это

`go run` компилирует Go-код во временный бинарный файл и сразу запускает его.

```bash
go run .
go run main.go
```

## Зачем нужно

Команда удобна во время обучения и быстрой проверки программы без ручной сборки бинарника.

## Основные варианты

```bash
go run .
go run ./cmd/app
go run . arg1 arg2
```

`go run .` запускает текущий package. Если это исполняемая программа, в пакете должны быть `package main` и `func main()`.

## Ключевые правила

- Для проекта с несколькими файлами чаще используй `go run .`, а не `go run main.go`.
- Аргументы после `.` передаются программе.
- Если запускаешь пакет из другой папки, укажи путь к package.
- `go run` не оставляет готовый бинарник в проекте.

## Частые ошибки

- запускать один файл, хотя код зависит от других файлов пакета;
- запускать пакет без `func main`;
- забывать, что аргументы после `go run .` идут в программу;
- путать ошибку компиляции и ошибку выполнения.
