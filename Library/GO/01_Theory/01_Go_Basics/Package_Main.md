# Package Main

## Что это

`package main` — специальный пакет, из которого Go собирает исполняемую программу.

```go
package main
```

Если файл принадлежит `package main` и в пакете есть функция `main`, такой код может стать запускаемым приложением.

## Зачем нужно

`package main` используется в точках входа:

- CLI-приложение;
- HTTP-сервер;
- worker-процесс;
- миграционная утилита;
- маленькая учебная программа.

## Где обычно лежит

В простом учебном проекте `main.go` может лежать в корне.

В более структурированном проекте часто используют папку `cmd`:

```text
cmd/
  app/
    main.go
```

## Ключевые правила

- В одном каталоге все `.go` файлы должны иметь один package.
- `package main` не стоит превращать в место для всей бизнес-логики.
- В `main` лучше оставить сборку зависимостей и запуск приложения.
- Повторяемую логику лучше выносить в отдельные пакеты.

## Частые ошибки

- смешать `package main` и другой package в одной папке;
- положить всю логику проекта в `main.go`;
- ожидать, что пакет без `func main` запустится как приложение;
- путать имя модуля из `go.mod` и имя package.
