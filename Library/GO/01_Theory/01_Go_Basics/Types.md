# Types

## Что это

Тип описывает, какие значения может хранить переменная и какие операции с этими значениями допустимы.

Go — статически типизированный язык. Это значит, что типы проверяются во время компиляции, до запуска программы.

## Основные группы типов

| Группа | Примеры | Для чего |
|---|---|---|
| Числа | `int`, `int64`, `float64` | счетчики, суммы, вычисления |
| Строки | `string` | текст |
| Логические | `bool` | условия и флаги |
| Составные | array, slice, map, struct | группы данных |
| Ссылочные | pointer, slice, map, channel, function | значения, которые могут быть `nil` |
| Интерфейсы | `interface`, `any` | поведение и абстракции |

## Выведение типа

Компилятор может вывести тип из значения:

```go
name := "Go"
count := 10
enabled := true
```

Но тип все равно существует. `count` не становится “любым числом”, это конкретный тип, обычно `int`.

## Преобразование типов

Go не делает опасные неявные преобразования между разными числовыми типами.

```go
var a int = 10
var b int64 = int64(a)
```

## Ключевые правила

- Проверяй тип на этапе проектирования данных.
- Не используй `string` для всего подряд.
- Для денег не выбирай `float64` без понимания последствий округления.
- Для внешних JSON-структур явно описывай поля через `struct`.
- Не используй `any`, если можно описать конкретный тип.

## Частые ошибки

- смешивать `int` и `int64` без явного преобразования;
- ожидать, что Go сам преобразует число в строку;
- хранить структурированные данные в `map[string]any`, когда нужна структура;
- забывать про `nil` у slice, map, pointer и interface.
