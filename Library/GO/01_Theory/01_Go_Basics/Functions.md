# Functions

## Что это

Функция — это именованный блок кода, который может принимать параметры, выполнять действие и возвращать результат.

В Go функция объявляется через `func`.

```go
func add(a int, b int) int {
    return a + b
}
```

## Зачем нужно

Функции помогают:

- разбивать программу на понятные шаги;
- переиспользовать логику;
- изолировать проверку ошибок;
- писать тестируемый код;
- не превращать `main` или HTTP-handler в длинный сценарий.

## Параметры и результат

Если несколько параметров имеют один тип, запись можно сократить:

```go
func add(a, b int) int {
    return a + b
}
```

Функция может возвращать несколько значений. Частый Go-паттерн — вернуть результат и ошибку:

```go
func findUser(id int) (User, error) {
    // пример формы, не готовая реализация
}
```

## Ключевые правила

- Функция должна делать одну понятную вещь.
- Чем меньше функция, тем проще ее читать и тестировать.
- Ошибку лучше возвращать явно, а не прятать внутри.
- Не стоит передавать слишком много параметров: это сигнал подумать о структуре.
- Название функции должно начинаться с действия: `createUser`, `parseConfig`, `sendResponse`.

## Частые ошибки

- писать всю логику в `main`;
- создавать функции с неясными именами вроде `process`;
- возвращать ошибку, но не добавлять полезный контекст;
- делать функцию, которая одновременно читает HTTP-запрос, ходит в БД и форматирует ответ.
