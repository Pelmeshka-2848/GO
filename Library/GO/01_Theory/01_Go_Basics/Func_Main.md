# Func Main

## Что это

`func main()` — точка входа исполняемой Go-программы.

```go
func main() {
    fmt.Println("Hello, Go")
}
```

Когда запускается compiled binary или `go run`, выполнение начинается с `main`.

## Зачем нужно

`main` отвечает за старт приложения:

- прочитать конфигурацию;
- собрать зависимости;
- запустить CLI-команду, HTTP-сервер или worker;
- корректно завершить программу при ошибке.

## Что не стоит делать в main

`main` не должен содержать всю бизнес-логику. Если он растет, код сложнее тестировать и сопровождать.

Лучше, когда `main` выглядит как сценарий сборки приложения, а основная логика находится в функциях и пакетах.

## Ключевые правила

- `func main()` не принимает аргументы и ничего не возвращает.
- Ошибки в `main` обычно логируются и приводят к завершению программы.
- В `main` допустимо связывать зависимости, но не стоит держать сложные алгоритмы.
- Для CLI аргументы читаются через `os.Args` или пакет `flag`.

## Частые ошибки

- пытаться вернуть ошибку из `main`;
- писать всю программу внутри `main`;
- не обрабатывать ошибку запуска сервера;
- смешивать чтение конфигурации, бизнес-логику и вывод ответа в одном блоке.
