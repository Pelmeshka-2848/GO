# Errors

## Что это

Ошибка в Go — это обычное значение, которое сообщает, что операция не смогла выполниться нормально.

Обычно ошибка возвращается последним результатом функции:

```go
result, err := doSomething()
if err != nil {
    return err
}
```

## Зачем нужно

Явная работа с ошибками делает код предсказуемым. В Go нет исключений как основного механизма обработки ошибок, поэтому разработчик сам решает, что делать с каждой ошибкой.

## Создание ошибок

Для простых ошибок используется `errors.New`:

```go
err := errors.New("empty name")
```

Для ошибки с контекстом часто используют форматирование:

```go
return fmt.Errorf("read config: %w", err)
```

`%w` сохраняет исходную ошибку для последующей проверки через `errors.Is` или `errors.As`.

## Ключевые правила

- Проверяй ошибку сразу после вызова, который может ее вернуть.
- Не используй `_`, если функция вернула ошибку.
- Добавляй контекст на границах слоев.
- Не показывай внутренние технические ошибки пользователю напрямую.
- Не логируй одну и ту же ошибку на каждом уровне.

## Частые ошибки

- игнорировать `err`;
- возвращать слишком общий текст;
- терять исходную ошибку при оборачивании;
- использовать panic для обычных ожидаемых ошибок.
