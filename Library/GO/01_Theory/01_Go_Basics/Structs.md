# Structs

## Что это

Структура — это пользовательский тип, который объединяет несколько полей в одно значение.

```go
type User struct {
    ID    int
    Name  string
    Email string
}
```

## Зачем нужно

Структуры нужны, чтобы описывать данные предметной области: пользователя, задачу, заказ, конфигурацию, HTTP-ответ.

## Создание значения

```go
user := User{
    ID:   1,
    Name: "Anna",
}
```

Лучше явно указывать имена полей. Такой код устойчивее к изменению порядка полей.

## Экспорт полей

Поле с заглавной буквы экспортируется:

```go
type User struct {
    Name string
}
```

Поле со строчной буквы доступно только внутри пакета:

```go
type user struct {
    name string
}
```

Для JSON, HTTP API и использования из других пакетов обычно нужны экспортируемые поля.

## Ключевые правила

- Структура должна описывать одну понятную сущность.
- Не превращай структуру в контейнер для случайных данных.
- Для JSON используй теги полей, если внешний формат отличается от Go-имен.
- Не добавляй поля “на будущее”, пока они не нужны.

## Частые ошибки

- забыть экспортировать поле и удивиться, что JSON пустой;
- использовать `map[string]any` вместо структуры;
- смешивать в одной структуре модель БД, API-ответ и внутреннюю бизнес-модель без причины;
- передавать большие структуры по значению, когда нужен указатель.
