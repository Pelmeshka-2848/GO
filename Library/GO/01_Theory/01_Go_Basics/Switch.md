# Switch

## Что это

`switch` выбирает одну из нескольких веток выполнения.

```go
switch operation {
case "add":
    fmt.Println("addition")
case "sub":
    fmt.Println("subtraction")
default:
    fmt.Println("unknown operation")
}
```

## Особенности Go

В Go после каждого `case` не нужен `break`: переход к следующему case не происходит автоматически.

Если нужно принудительно провалиться в следующую ветку, есть `fallthrough`, но в обычном коде он нужен редко.

## Switch без выражения

`switch` может работать как более читаемая цепочка `if`.

```go
switch {
case score >= 90:
    grade = "A"
case score >= 70:
    grade = "B"
default:
    grade = "C"
}
```

## Зачем нужно

`switch` удобен для:

- выбора CLI-операции;
- обработки HTTP-метода;
- обработки enum-подобных статусов;
- разветвления по типу ошибки или события.

## Ключевые правила

- Используй `default` для неизвестного значения.
- Не злоупотребляй `fallthrough`.
- Если ветки стали большими, вынеси логику в функции.
- Для бизнес-статусов лучше использовать именованные константы.

## Частые ошибки

- ожидать поведение как в C/JavaScript с обязательным `break`;
- забыть `default`;
- держать много сложной логики внутри case;
- сравнивать строки, которые лучше заменить константами.
