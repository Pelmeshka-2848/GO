# Maps

## Что это

Map — это структура данных “ключ-значение”.

```go
ages := map[string]int{
    "Anna": 30,
    "Ivan": 25,
}
```

Ключ должен быть сравнимым типом. Значение может быть почти любого типа.

## Зачем нужно

Map удобна, когда нужно быстро находить значение по ключу:

- настройки по имени;
- кэш по ID;
- счетчик частот;
- таблица соответствий;
- временное хранилище в учебном API.

## Чтение значения

```go
age, ok := ages["Anna"]
if !ok {
    // ключ не найден
}
```

Флаг `ok` нужен, чтобы отличить отсутствующий ключ от нулевого значения.

## Создание через make

```go
users := make(map[int]string)
users[1] = "Anna"
```

Запись в nil map вызовет panic, поэтому map нужно инициализировать.

## Ключевые правила

- Проверяй наличие ключа через `value, ok`.
- Не полагайся на порядок обхода map.
- Инициализируй map перед записью.
- Для сложных данных часто лучше использовать struct как значение.

## Частые ошибки

- писать в nil map;
- ожидать стабильный порядок при `range`;
- не отличать “ключа нет” от “значение равно нулю”;
- использовать map вместо нормальной структуры данных.
