# Constants

## Что это

Константа — это именованное значение, которое известно на этапе компиляции и не меняется во время выполнения программы.

```go
const appName = "taskflow"
const defaultPort = 8080
```

## Когда использовать

- фиксированные строки;
- числовые лимиты;
- значения по умолчанию;
- имена ролей, статусов, режимов;
- повторяемые литералы, смысл которых важен для чтения кода.

## Типизированные и нетипизированные константы

В Go константа может быть без явно заданного типа:

```go
const timeoutSeconds = 30
```

Компилятор подберет подходящий тип в месте использования, если это безопасно.

Можно задать тип явно:

```go
const maxRetries int = 3
```

## Ключевые правила

- Не выноси в константу значение, которое должно приходить из конфигурации.
- Константа должна улучшать смысл кода, а не просто прятать число.
- Для связанных значений можно использовать блок `const`.
- Для набора статусов лучше подумать о собственном типе.

## Частые ошибки

- хранить секреты в константах;
- использовать константы для env-specific настроек;
- создавать константу с неясным именем вроде `Value1`;
- путать константы с переменными конфигурации.
