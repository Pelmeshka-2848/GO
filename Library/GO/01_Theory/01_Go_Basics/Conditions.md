# Conditions

## Что это

Условия позволяют выполнять разные ветки кода в зависимости от логического выражения.

```go
if age >= 18 {
    fmt.Println("adult")
} else {
    fmt.Println("minor")
}
```

## Особенности Go

В Go условие не пишется в круглых скобках. Фигурные скобки обязательны.

Перед условием можно объявить короткую переменную:

```go
if err := validate(); err != nil {
    return err
}
```

Такая переменная видна только внутри `if` и связанных `else`.

## Зачем нужно

Условия используются для:

- проверки ошибок;
- валидации входных данных;
- выбора бизнес-правила;
- обработки разных HTTP-статусов;
- защиты от некорректных состояний.

## Ключевые правила

- Сначала обрабатывай ошибочные и граничные случаи.
- Не делай слишком глубокую вложенность.
- Если условие стало большим, вынеси его в функцию с понятным именем.
- Не сравнивай boolean с `true` или `false` без необходимости.

## Частые ошибки

- забыть `else` там, где ветки должны быть взаимоисключающими;
- спрятать важную ошибку глубоко внутри вложенных `if`;
- использовать слишком сложное условие без объясняющего имени;
- путать `=` и `==`.
