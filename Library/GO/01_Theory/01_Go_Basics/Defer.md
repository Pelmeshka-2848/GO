# Defer

## Что это

`defer` откладывает выполнение функции до момента выхода из текущей функции.

```go
file, err := os.Open("data.txt")
if err != nil {
    return err
}
defer file.Close()
```

## Зачем нужно

`defer` используют для освобождения ресурсов:

- закрыть файл;
- закрыть соединение;
- снять lock;
- завершить измерение времени;
- выполнить cleanup перед выходом.

## Порядок выполнения

Если несколько `defer` объявлены в одной функции, они выполняются в обратном порядке: последний объявленный выполнится первым.

```go
defer fmt.Println("first")
defer fmt.Println("second")
// сначала second, потом first
```

## Важная деталь

Аргументы deferred-функции вычисляются в момент объявления `defer`, а не в момент выполнения.

## Ключевые правила

- Ставь `defer` сразу после успешного получения ресурса.
- Не используй `defer` до проверки ошибки открытия ресурса.
- Помни про обратный порядок выполнения.
- В горячих циклах `defer` может быть не лучшим выбором из-за накопления отложенных вызовов.

## Частые ошибки

- вызвать `defer file.Close()` до проверки `err`;
- использовать `defer` внутри большого цикла без понимания последствий;
- ожидать прямой порядок выполнения нескольких `defer`;
- забыть, что значения аргументов фиксируются сразу.
