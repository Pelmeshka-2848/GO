# testing package

## Кратко

`testing` — стандартный пакет для написания unit-тестов, benchmarks и examples в Go.

## Зачем нужно

- проверять функции автоматически;
- фиксировать ожидаемое поведение;
- защищаться от регрессий;
- измерять производительность через benchmarks.

## Тестовые файлы

Файлы тестов заканчиваются на `_test.go`.

```go
func TestSomething(t *testing.T) {
    // проверка поведения
}
```

## Основные возможности

- `t.Fatal` и `t.Fatalf` — завершить тест сразу;
- `t.Error` и `t.Errorf` — отметить ошибку и продолжить;
- `t.Run` — subtests;
- `t.Helper` — пометить helper-функцию;
- `b.N` — цикл benchmark-а.

## Правила

- Тестируй поведение, а не внутренние шаги.
- Ошибочные сценарии так же важны, как успешные.
- Для набора похожих случаев используй table-driven tests.
- Тест должен быть воспроизводимым.

## Типичные ошибки

| Ошибка | Почему возникает | Как исправить |
|---|---|---|
| Проверять только happy path | Ошибки не покрыты | Добавить negative cases |
| Тест зависит от времени | Нестабильность | Использовать контролируемые значения |
| Большой setup в каждом тесте | Дублирование | Вынести helper |
