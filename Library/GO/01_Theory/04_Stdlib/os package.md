# os package

## Кратко

`os` — стандартный пакет для работы с операционной системой: файлы, аргументы командной строки, переменные окружения, завершение процесса.

## Зачем нужно

- читать `os.Args`;
- работать с файлами;
- читать env-переменные;
- создавать директории;
- завершать программу с кодом выхода.

## Частые функции

- `os.Args`;
- `os.Getenv`;
- `os.LookupEnv`;
- `os.ReadFile`;
- `os.WriteFile`;
- `os.Open`;
- `os.Create`;
- `os.MkdirAll`;
- `os.Exit`.

## Правила

- Проверяй ошибки файловых операций.
- Для обязательных env-переменных используй `LookupEnv`, чтобы отличать пустое значение от отсутствующего.
- Не вызывай `os.Exit` глубоко в бизнес-логике.
- Для путей между ОС используй `path/filepath`.

## Типичные ошибки

| Ошибка | Почему возникает | Как исправить |
|---|---|---|
| `os.Args[1]` без проверки | Аргумента может не быть | Проверять `len(os.Args)` |
| Игнор ошибки файла | Файл может не существовать | Проверять `err` |
| `os.Exit` в service | Слой убивает процесс | Вернуть ошибку наверх |
