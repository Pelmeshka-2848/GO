# errors package

## Кратко

`errors` — стандартный пакет для создания и проверки ошибок.

## Зачем нужно

- создавать простые ошибки;
- сравнивать ошибки через `errors.Is`;
- извлекать типизированные ошибки через `errors.As`;
- работать с error wrapping.

## Основные функции

- `errors.New` — создать ошибку с текстом;
- `errors.Is` — проверить, содержит ли цепочка ошибок конкретную ошибку;
- `errors.As` — найти ошибку конкретного типа в цепочке;
- `errors.Join` — объединить несколько ошибок.

## Основная идея

Ошибка в Go может быть обернута дополнительным контекстом. При правильном wrapping исходную причину можно не потерять.

```go
return fmt.Errorf("read config: %w", err)
```

## Правила

- Для простой статичной ошибки используй `errors.New`.
- Для добавления контекста используй `fmt.Errorf` с `%w`.
- Для проверки wrapped-ошибок используй `errors.Is`, а не прямое сравнение.
- Не сравнивай текст ошибки как контракт.

## Типичные ошибки

| Ошибка | Почему возникает | Как исправить |
|---|---|---|
| Сравнение текста ошибки | Текст легко меняется | Использовать sentinel error или тип |
| `%v` вместо `%w` | Исходная ошибка теряется для `errors.Is` | Использовать `%w` |
| Panic вместо error | Ожидаемую ошибку сделали аварией | Вернуть `error` |
